MOSS-VoiceGenerator OpenMOSS (Fudan University)
NL 描述生成 Apache 2.0

定位：MOSS 团队的专用音色生成模型，用自然语言描述生成语音，无需录音。1.7B，Apache 2.0。与 MOSS-TTS 互补。

代码示例

Python — NL 描述生成虚拟音色from transformers import AutoModel, AutoProcessor

processor = AutoProcessor.from_pretrained(
    "OpenMOSS-Team/MOSS-VoiceGenerator", trust_remote_code=True)
model = AutoModel.from_pretrained(
    "OpenMOSS-Team/MOSS-VoiceGenerator",
    trust_remote_code=True, device_map="cuda:0",
    torch_dtype=torch.bfloat16,
)

# instruction 为必需参数，自然语言描述生成音色
messages = [processor.build_user_message(
    text="A warm gentle middle-aged woman, slight southern accent, calming rhythm.",
    instruction="Generate a voice matching this description.",
)]
batch = processor([messages], mode="generation")
outputs = model.generate(
    input_ids=batch["input_ids"].to("cuda:0"),
    attention_mask=batch["attention_mask"].to("cuda:0"),
    max_new_tokens=2048,
)

按场景推荐最佳模型组合

Top 3 组合方案

  1. 🥇 CosyVoice 2/3 (0.5B) — 主力引擎。NL 指令 + 显式语速 + voice cloning + inline 副语言标签。Apache 2.0 完全开源，体积小部署快。
  2. 🥈 IndexTTS 2 (~1B) — 精细情感引擎。8 维连续情感向量 + 情感/音色解耦。与 CosyVoice 互补：CosyVoice 负责通用 + 语速，IndexTTS 2 负责精细情感。
  3. 🥉 Fish S2 Pro (~4.4B) — 高级场景引擎。亚词级标签 + 原生多角色 + 开放域 NL 描述。可控性天花板，用于多角色对话等杀手级场景。

注意事项

本次验证发现多处与先前认知不符：CosyVoice 的 inference_instruct2 需要 prompt_wav（非可选）；Chatterbox Turbo 的 exaggeration/cfg_weight 被忽略；MOSS-TTS 无显式情感参数但有 [pause] 标签；IndexTTS 2 无显式语速控制；dots.tts 无 "soar" 变体。所有代码示例均经源码验证。

SSML 支持现状

无开源 TTS 原生支持 W3C SSML 1.1。但 Fish S2 的 [tag]、CosyVoice 的 [breath]/[laughter]、Chatterbox Turbo 的 [laugh]、Dia 的 (laughs)、MOSS/IndexTTS 的拼音注解均为 SSML-like inline 语法。需自行编写解析层桥接。
