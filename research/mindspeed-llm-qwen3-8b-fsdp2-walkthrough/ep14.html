微调 vs 预训练差异

目标：讲清 tune = pretrain 去 mg 定长 + 换 alpaca 模板 + 关 shuffle。前置：EP13 数据流水线。

你将获得：① 两份 A2 YAML 的逐行 diff，精确到哪几行只属于预训练、哪几行只属于微调 ② 一个反直觉点——微调不改并行配置，fsdp_modules 与预训练逐字相同

你已经熟悉的情况 桥接开场

你做过单卡训练，最熟的二分是 train vs eval：train 开 dropout、requires_grad、反向；eval 关 dropout、torch.no_grad、只前向。配置层差异无非一个 model.eval() 切换，模型结构、数据加载器、优化器都不变。这是同一份代码的两种运行模式。

MindSpeed-LLM 的 FSDP2 后端里，预训练（pt）与微调（tune/SFT） 的关系很像——但差异不在运行模式开关，而在配置文件的几行 YAML。同一目录 examples/fsdp2/qwen3/ 下并排放着 pretrain_qwen3_8b_4k_fsdp2_A2.yaml 与 tune_qwen3_8b_4k_fsdp2_A2.yaml，命名只差一个前缀。把它们并排 diff，会发现真正变动的就那么四五行：stage、data_manager_type、dataset 来源、disable_shuffling、以及 num_train_epochs。并行配置（parallel 段）逐字不变——这是最容易被忽略、却最关键的一点。下面把这份 diff 摊开。

本项目怎么做 原理

微调不是"另起一套配置"，而是从预训练配置里删掉 mg 定长路径的两把钥匙（data_manager_type: mg 与 stage: pt）、换掉数据集来源（从预训练的 JSON/parquet 路径换成微调注册表里的 alpaca_full 模板）、关掉 shuffle（微调样本量小且要保证可复现）。删掉 mg 钥匙后，DataFactory 入口的 stage×data_manager_type 判定（见 EP13 的 data_factory.py）就自然落到 stage=sft 默认 + LFDataManager 在线渲染路径——alpaca 模板的 converter/template/collator 三段这才派上用场。并行段（fsdp_modules/ep_modules/ep_fsdp_modules/recompute）一个字都不动，因为微调改的是"喂什么数据、要不要 shuffle、训几轮"，不是"模型怎么切"。

核实说明：tune_qwen3_8b_4k_fsdp2_A2.yaml 未 显式写 disable_shuffling: 1（逐行读过，tune YAML 第 26–42 行的 training 段没有此键）。该值是 finetune_fsdp2.md 第 165 行列出的官方微调训练参数，含义为"是否禁用训练集洗牌"。微调样本量远小于预训练、且要求 loss 曲线可复现，故文档将其列为推荐项；具体 A2 配置是否落地该键，以源码为准——本 A2 未落地。读者迁移到自己数据集时应按需补此行。这与 brief 里"tune 有 disable_shuffling: 1"的表述存在出入，此处据实标注。

官方文档第 210 行一句话定性：data_manager_type 的 lf=微调、mg=预训练——这正是 A2 两份 YAML 差异的"官方释义"。第 204 行给预训练示例 dataset 直接填 parquet 路径、第 73 行给微调示例 dataset 填注册名（alpaca_full）支持逗号分隔混合，与 A2 sh 的 --data.dataset alpaca_full 一致。文档把"微调走 lf+注册名、预训练走 mg+原始路径"讲得最清楚，本篇 A2 配置是该规则的实例。

预训练 pretrain (pt)

data_manager_typemg —— 走 MegatronDataManager，离线 .bin/.idx 定长
stagept —— 预训练阶段，与 mg 配对（data_factory.py 硬约束）
disable_shuffling默认（不显式写）—— 训练集洗牌开
dataset 来源YAML 内联 JSON 路径 / 原始 parquet 路径（不经注册表）
num_train_epochs3.0（与 max_steps=2000 并存，后者实际主导）
并行段fsdp_modules 等 —— 与微调逐字相同

微调 tune (sft)　（本篇特殊：非 GPU/NPU 对照）

data_manager_type无（省略）—— 默认 lf，走 LFDataManager 在线渲染
stage无（省略）—— 默认 sft，走 alpaca 模板三段
disable_shuffling文档推荐 1（A2 未显式落地，以源码为准）
dataset 来源CLI --data.dataset alpaca_full —— 注册表 key（dataset_info.json）
num_train_epochs无（删之）—— 仅 max_steps=2000 主导步数
并行段fsdp_modules 等 —— 与预训练逐字相同
小结　心智模型：tune = pretrain 删 data_manager_type: mg + 删 stage: pt（让入口落到 lf/sft 在线路径）+ 删 num_train_epochs（让 max_steps 主导）+ 换数据来源（YAML 路径 → CLI alpaca_full 注册名）+ 文档推荐关 shuffle；并行配置（fsdp_modules/ep_*/recompute）一字不改，因为微调改的是"喂什么数据、要不要洗牌、训几步"，而非"模型怎么切"。
常见误区：① 以为微调要改并行配置——fsdp_modules 与预训练相同。实测两份 A2 YAML 的 parallel 段（L13–25）逐字一致，微调只动 data/training 两段，不改并行；若误改 fsdp_modules 会破坏与预训练 checkpoint 的分片对齐，加载权重会错位。② 以为 disable_shuffling 是 A2 tune YAML 里写死的行——实则 A2 未落地该键，仅 finetune_fsdp2.md L165 列为推荐训练参数，迁移时按需补。③ 以为 alpaca_full 是文件名——它是 dataset_info.json 注册表 key，背后 file_name 才指向真实 parquet；预训练才直接在 dataset 填路径。④ 以为删 stage 会报错——stage 省略时落到默认 sft，与省略的 data_manager_type（默认 lf）天然配对，是文档鼓励的微调写法。
