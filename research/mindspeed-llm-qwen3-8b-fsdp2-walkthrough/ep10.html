通信与计算掩盖

目标：讲清 HCCL premul_sum、Prefetch、Zero-Copy 与分片梯度范数聚合。前置：EP09 专家并行。

你将获得：① 一条从 train_fsdp2.py:112 在 NPU 分支把 torch.distributed.all_reduce/reduce_scatter 三处替换成 hccl_premul_sum_wrapper、让 HCCL 在 reduce 阶段就完成梯度缩放（省一次 kernel 启动）的 patch 链路，再到 clip_grad_norm.py 把 DTensor 分片梯度本地算 norm**p 后跨 fsdp/ep_fsdp/ep 多组 all_reduce 聚合出全局范数的完整落地 ② 弄懂"通信掩盖"在 NPU 侧由谁负责——apply_hccl_premul_sum_patch 与 clip_grad_norm 在仓内、TORCH_HCCL_ZERO_COPY=1 是环境变量、而 Prefetch 的真正重叠逻辑在 fsdp_turbo/torch FSDP2（仓内只见配置透传）——配 A3.sh 三项融合算子开关把 kernel 调用次数压到最低

你已经熟悉的情况 桥接开场

你在 GPU 侧练过 DDP/FSDP：反向算完梯度后 torch.distributed.all_reduce 跨卡求和、再 clamp 做梯度裁剪，通信是 NCCL 的 ReduceOp.SUM；想缩放梯度（比如梯度除以 world_size 或乘 loss scale），常在 reduce 之后单独 mul_ 一次——多一次 kernel 启动。你还用过 FSDP 的"通信计算掩盖"：PyTorch 在 fully_shard 的反向里把当前层的 reduce-scatter 和下一层的反向计算用不同 stream 重叠起来，让通信和算力并发跑。这一层"掩盖"的实现散落在 torch 的 FSDP2 unshard/prefetch 逻辑里，对你是黑盒。到了 NPU 这边，通信后端从 NCCL 换成 HCCL（华为集合通信库），上面的两条需求（reduce 期缩放、通信计算重叠）都要重新落地——MindSpeed-LLM 的做法是：patch 掉裸 all_reduce 让它支持 PREMUL_SUM（pre-multiplied sum，reduce 期乘系数），配置透传 prefetch 参数到 FSDP2 实现，环境变量开 TORCH_HCCL_ZERO_COPY 让通信 buffer 免拷贝，再把 FSDP2 分片梯度范数的跨分片聚合自己重写一遍（torch 原生不直接支持 EP 混合组）。本篇就讲清这四件事各自落在哪个文件、哪些是仓内可见、哪些是委派给外部。

本项目怎么做 原理

通信掩盖的核心诉求是把每次反向里的两类工作——"通信"（all-gather 拼参数 / reduce-scatter 散梯度 / all-reduce 聚范数）与"计算"（matmul、norm、kernel）——并发跑，别让 NPU 干等通信。NPU 侧的掩盖分三层：① 通信后端层，HCCL 替代 NCCL，并通过 apply_hccl_premul_sum_patch 给 all_reduce 增加"reduce 期预乘系数"能力——梯度除以 world_size、loss-scale 反除、EP 梯度除数（EP09 讲的 ep*efsdp*edp factor）这类缩放，原本要 reduce 完再单独 mul_，patch 后在 reduce 内核里一次完成；② FSDP2 流水线层，前向 prefetch（算第 L 层时异步 all-gather 第 L+1 层参数）、反向 prefetch（reduce-scatter 第 L 层梯度时异步拉第 L-1 层全参），让"下一层通信"与"当前层计算"用两个 stream 重叠——这部分的重叠调度逻辑不在本仓（EP08 已述 dense 路径的 fully_shard 实现在外部 fsdp_turbo 包），仓内只见 num_to_forward_prefetch / num_to_backward_prefetch 两个透传参数；③ buffer 零拷贝层，export TORCH_HCCL_ZERO_COPY=1 让 HCCL 通信 buffer 与框架 tensor 共享底层显存、省去通信前的 host↔device 或 device↔device 拷贝。再把分片梯度范数聚合（clip_grad_norm）单独算一笔账：FSDP2 下每个参数的梯度是 DTensor（本卡只持 1/N 片），算局部 norm**p 后需跨 fsdp 组 all-reduce 求和、再开 p 次方——EP 模型还要把 ep/efsdp 两组也算进来。下面的 SVG 把这三层与一次反向的时序画在一起。

这张图的诚实之处在于把"在仓内"与"委派外部"分得很清：① 通信后端 patch（torch_patch.py 经 train_fsdp2.py:112 调用）和 ④ 梯度范数聚合（clip_grad_norm.py）是仓内可见、可逐行读的实现；③ TORCH_HCCL_ZERO_COPY=1 是环境变量（A3.sh:2），生效与否在 HCCL 运行时；② Prefetch 的透传参数在仓内（model_factory.py:168-169），但真正的"下一层通信与当前层计算重叠"的调度逻辑在外部 fsdp_turbo / torch FSDP2（EP08 已述 dense 路径 fully_shard 在 fsdp_turbo，本仓无 prefetch 重叠源码）——所以图里 ② 这一条是按 torch FSDP2 的 prefetch 语义画的，不是贴仓内某行代码。⑤ 是 CLI 开关 + modeling 层的 fused 算子，与通信掩盖是"另一条减 kernel 数"的互补路径。

三行 torch.distributed.xxx = wrapper(xxx, ...) 是猴子补丁——直接替换全局函数指针。被包的三处原语恰好覆盖 FSDP2 反向的通信骨架：all_reduce（梯度范数聚合、loss 缩放）、reduce_scatter / reduce_scatter_tensor（FSDP 梯度分片散回）。第二个参数 'tensor' / 'output' 是各原语"输出张量"的 kwarg 名——wrapper 要靠它在 wait 之后定位输出 tensor 做 mul_。

hccl_premul_sum_wrapper：拦截 PREMUL_SUM、reduce 后乘系数（行 5-18）
def hccl_premul_sum_wrapper(op, output_name):
    def wrapper(*args, **kwargs):
        factor = None
        if 'op' in kwargs and kwargs['op'] == ReduceOp.PREMUL_SUM:
            factor = kwargs['op'].__getstate__()[1]
            kwargs['op'] = ReduceOp.SUM
        handle = op(*args, **kwargs)
        if handle is not None:
            handle.wait()
        if factor is not None:
            output = args[0] if len(args) > 0 else kwargs[output_name]
            output.data.mul_(factor)
        return handle
    return wrapper
见 ../_deps_mindspeed_core/mindspeed/fsdp/utils/torch_patch.py：:5（def hccl_premul_sum_wrapper(op, output_name)——闭包工厂）、:8（if kwargs['op'] == ReduceOp.PREMUL_SUM——只对 PREMUL_SUM 这种 reduce-with-scale 起作用，普通 SUM/MAX 不动）、:9（factor = kwargs['op'].__getstate__()[1]——从 ReduceOp 状态里取出预乘系数，PREMUL_SUM 的语义即 "sum then multiply by factor"）、:10（kwargs['op'] = ReduceOp.SUM——把 PREMUL_SUM 降级成普通 SUM，让底层 HCCL 认得）、:11（handle = op(*args, **kwargs)——调用原 all_reduce/reduce_scatter）、:13（handle.wait()——同步等通信完成）、:15（output = args[0] if len(args)>0 else kwargs[output_name]——定位输出 tensor，all_reduce 是位置参数 'tensor'，reduce_scatter 是 kwarg 'output'）、:16（output.data.mul_(factor)——reduce 完就地把结果乘系数，省去单独一次 mul kernel）、:17（return handle——保持原返回签名）

这是 premul_sum 的核心。PREMUL_SUM 是一种带系数的 sum-reduce：语义上等价于"先求和、再每个元素乘 factor"——常见用途是梯度除以 world_size、loss scale 反除、EP 梯度除数（EP09 的 ep*efsdp*edp factor）。torch 在新版本把 PREMUL_SUM 做进了 ReduceOp，但HCCL 当时不一定原生支持——这个 wrapper 的作用就是在 Python 层把 PREMUL_SUM 拆成 "SUM + 事后 mul_"：先取系数、把 op 降级成 SUM 调底层 HCCL、wait 完再 output.data.mul_(factor)。注意行 16 的 mul_ 是原地乘、且在 wait() 之后——意味着 reduce 结果落盘后立刻乘，省一次独立 kernel 启动（虽然 mul 本身仍在 reduce 之后，但比"reduce→return→用户再调 mul"少一次往返）。何时触发 PREMUL_SUM？看上层谁传了 op=ReduceOp.PREMUL_SUM——torch FSDP2 在梯度 reduce 时会带系数（如 1/world_size），本仓的 clip_grad_norm 用的是普通 SUM（见下行 232），不触发这条 premul 分支，走的是 wrapper 里 factor=None 的直通路径。

def clip_grad_norm(
    model, max_norm: float, norm_type: float = 2.0, error_if_nonfinite: bool = False, foreach: bool | None = None
) -> torch.Tensor:
    # EP-aware path (FSDP2 + EP): maintain mathematical parity with FSDP1 clipper
    if hasattr(model, "_ep_param_groups"):
        return ep_fsdp2_clip_grad_norm(model, max_norm, norm_type=norm_type,
            error_if_nonfinite=error_if_nonfinite, foreach=foreach)

    grad_norm = fsdp2_clip_grad_norm(model.parameters(), max_norm, norm_type=norm_type,
        error_if_nonfinite=error_if_nonfinite, foreach=foreach)
    if isinstance(grad_norm, DTensor):
        grad_norm = grad_norm.full_tensor()
    return grad_norm

fsdp2_clip_grad_norm：DTensor 与普通 Tensor 严格分流，分别算范数再合并（行 42-104）
@torch.no_grad()
def fsdp2_clip_grad_norm(parameters, max_norm, norm_type=2.0, error_if_nonfinite=False, foreach=None):
    # ... separate DTensor and native Tensor grads ...
    for p in parameters:
        if p.grad is not None:
            if isinstance(p.grad, DTensor):
                dtensor_params.append(p); dtensor_grads.append(p.grad)
            else:
                tensor_params.append(p); tensor_grads.append(p.grad)

    if len(dtensor_grads) > 0:
        norm_d = torch.nn.utils.get_total_norm(dtensor_grads, norm_type, error_if_nonfinite, foreach)
        if isinstance(norm_d, DTensor):
            norm_d = norm_d.full_tensor()
    else:
        norm_d = torch.tensor(0.0, device=parameters[0].device if parameters else 'cpu')
    # ... same for native tensor_params → norm_t ...
    if norm_type >= math.inf:
        total_norm = torch.max(norm_d, norm_t)
    else:
        total_norm = (norm_d**norm_type + norm_t**norm_type) ** (1.0 / norm_type)

    # Context Parallel (CP) scale handling
    ps = ParallelState()
    total_norm *= ps.get_group_size("cp")
    # Execute clipping separately to prevent foreach_mul operator crash
    if len(dtensor_params) > 0:
        torch.nn.utils.clip_grads_with_norm_(dtensor_params, max_norm, total_norm, foreach)
    if len(tensor_params) > 0:
        torch.nn.utils.clip_grads_with_norm_(tensor_params, max_norm, total_norm, foreach)

def ep_fsdp2_clip_grad_norm(model, max_norm, norm_type=2.0, error_if_nonfinite=False, foreach=None):

    ep_group = ps.get_ep_group() if ps.is_ep_enable() else None
    ep_fsdp_group = None

    ep_params = [p for p in model._ep_param_groups.get("ep", []) if p.grad is not None]
    non_ep_params = [p for p in model._ep_param_groups.get("non_ep", []) if p.grad is not None]

    non_ep_total = _fsdp2_reduce_group(params=non_ep_params, norm_type=norm_type,
        reduce_groups=[("fsdp", fsdp_group)])
    ep_total = _fsdp2_reduce_group(params=ep_params, norm_type=norm_type,
        reduce_groups=[("ep_fsdp", ep_fsdp_group), ("ep", ep_group)])

    if math.isinf(norm_type):
        total_norm = torch.maximum(non_ep_total, ep_total)
    else:
        total_norm = (non_ep_total + ep_total) ** (1.0 / float(norm_type))

    torch.nn.utils.clip_grads_with_norm_(non_ep_params, max_norm, total_norm, foreach=False)
    return total_norm
