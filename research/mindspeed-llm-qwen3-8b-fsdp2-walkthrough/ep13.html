数据流水线

目标：理清 raw 数据→formatter→converter→template→collator→batch 的流水。前置：EP12 Qwen3 建模精读。

你将获得：① 一条从 parquet 原始样本到 FSDP2 训练 batch 的完整链路地图 ② 预训练（mg 定长）与微调（alpaca 短样本）两条路径的分叉点，以及 megatron_data/ 目录存在的真实理由

你已经熟悉的情况 桥接开场

你写过 PyTorch 训练循环，三件套是肌肉记忆：Dataset 负责按索引吐出一条样本、DataLoader 负责多进程拉取与拼批、collate_fn 负责把一列表样本对齐成张量 batch。分布式再加一个 DistributedSampler，按 rank 切分索引、保证各卡不重叠。

这套够用，但只解决了"喂张量"这一段。语言模型训练还有两个它管不了的问题：原始样本是 prompt+response 的对话，不是 token id；且各样本长短不一，直接拼批会因 padding 浪费算力，更无法表达"只对 response 计 loss"。于是本项目的 fsdp2 data 模块在标准三件套前面，再插了三段——formatter（按模板把对话填成字符串）、converter（把原始列名对齐成统一 _prompt/_response 结构）、template（qwen3 的 chat 模板，定义 <|im_start|> 等特殊标记摆位），后面再接一个支持 4D 注意力掩码的 collator。此外，预训练场景数据量大、要求定长连续拼接，于是另起一条 mg（Megatron）路径。下面把这条流水一段段拆开看。

本项目怎么做 原理

整条流水是一个"格式化→转模板→拼批"的三段串接，外加一个在入口处的二选一分叉。入口 DataFactory.create 看 training_args.stage 与 data_manager_type 两个开关，二选一返回 LFDataManager（llamafactory 风格，微调用）或 MegatronDataManager（Megatron 风格，预训练用）。LF 路径里，get_dataset 先加载原始 parquet，经 align_dataset 用 converter 把列名对齐成 _prompt/_response/_system/_tools 标准结构，再用 _get_dataset_processor 调用 template 把它 token 化成 input_ids/labels（label 里 prompt 部分 mask 成 IGNORE_INDEX）。最后 SFTDataCollatorWith4DAttentionMask 按批 padding 并在 neat_packing 时生成块对角 4D 掩码。mg 路径则走 train_valid_test_datasets_provider，读 mcore 的 indexed_dataset（.bin/.idx 定长 mmap），用 MegatronPretrainingSampler 按定长步长切样本，不经 template 在线拼。

mindspeed_llm/fsdp2/data/collator.py — SFTDataCollatorWith4DAttentionMask：在父类 padding 之上，neat_packing 时生成块对角 4D 掩码

def prepare_4d_attention_mask(attention_mask_with_indices, dtype):
    # (batch, seq) 索引掩码 → (batch, 1, seq, seq) 下三角块对角掩码
    non_padding_mask = (attention_mask_with_indices != 0).unsqueeze(1).unsqueeze(2)
    indices = attention_mask_with_indices.unsqueeze(1).unsqueeze(2)
    indices_t = attention_mask_with_indices.unsqueeze(1).unsqueeze(3)
    tril_mask = torch.tril(torch.ones((seq_len, seq_len), dtype=torch.bool))
    attention_mask_4d = (indices == indices_t) & non_padding_mask & tril_mask
    return torch.where(attention_mask_4d, zero_tensor, min_dtype)

@dataclass
class SFTDataCollatorWith4DAttentionMask(DataCollatorForSeq2Seq):
    def __call__(self, features):
        features = super().__call__(features)
        if self.block_diag_attn and self.attn_implementation != "flash_attention_2":
            features["attention_mask"] = prepare_4d_attention_mask(features["attention_mask"], self.compute_dtype)
        return features

mindspeed_llm/fsdp2/data/collator.py:10（prepare_4d_attention_mask） · :54（SFTDataCollatorWith4DAttentionMask） · :61（__call__ 触发块对角掩码）

职责一句话：collator 负责"把一列表变长样本对齐成定张量 batch，并在 neat_packing（把多条短样本打包进同一序列）时生成 4D 块对角掩码防止样本间互相 attend"。它继承自 transformers 的 DataCollatorForSeq2Seq，padding 到 cp_size*2 的倍数（data_factory.py 第 88 行），label 的 pad 位填 IGNORE_INDEX。

为何存在：mcore 的预训练数据是离线预处理好的一套 indexed 格式（每个样本已是定长 token id 序列，.bin 存数据、.idx 存索引），fsdp2 后端为了复用这套既有预处理产物与 MegatronPretrainingSampler 的定长步长切片逻辑，原样保留了 megatron_data/ 整个目录。它本质是"mcore 数据格式兼容层"，不是新的数据流。不在本篇深挖实现，只定位其存在理由。

GPU / 已知

Datasettorch.utils.data.Dataset 按索引吐样本，是"喂张量"的最小单位
DataLoader单机多进程拉取+拼批，配 RandomSampler/DistributedSampler
collate_fn用户手写或 DataCollatorForSeq2Seq，做 padding 与 label 对齐
templateHF ChatTemplate / apply_chat_template，jinja 渲染对话

NPU / 本项目

DatasetLF 路径 get_dataset 在线渲染（converter+template）；mg 路径走 megatron_data indexed .bin/.idx，自带 mcore Dataset
DataLoader同一 torch DataLoader，但 sampler 分叉：LF 用 DistributedSampler（dp_fsdp rank/size），mg 用 MegatronPretrainingSampler（按定长步长切）
collate_fnSFTDataCollatorWith4DAttentionMask：neat_packing 时额外生成 4D 块对角掩码（兼容 NPU eager attention，无 FA2 时启用）
template自研 Template 体系：register_template 注册 qwen3，StringFormatter 槽位填 {{content}}，ReasoningTemplate 支持推理开关
小结　心智模型：数据 = 格式化（converter 抹平列名）→ 转模板（template/formatter 把对话填进 ChatML 槽位并 tokenize、prompt 部分 mask 成 IGNORE_INDEX）→ 拼批（collator padding + 4D 块对角掩码）三段串接；入口 DataFactory 用 stage×data_manager_type 二选一，LF 在线渲染走 sft，mg 走 mcore 定长 indexed 走 pt。
常见误区：① 以为预训练(mg定长)与微调(alpaca)数据路径相同——实则两条配置一个带 data_manager_type: mg+stage: pt 走 MegatronDataManager 的离线定长 indexed，一个不带这两个键默认落到 LFDataManager 的在线 alpaca 渲染，converter/template/collator 三段只服务 LF 路径。② 以为 megatron_data/ 是新数据流——它只是为兼容 mcore 已有的 .bin/.idx 预处理产物而保留的格式兼容层。③ 以为 template 只渲染字符串不管 token——其实 get_dataset 流程里 template 同时负责 tokenize 与 label mask（IGNORE_INDEX），formatter 才是只产字符串的那一层。
