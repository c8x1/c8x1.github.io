FSDP2 分片落地

目标：讲清 fsdp_modules 列表如何变成 fully_shard 原位调用。前置：EP07 并行状态与引擎。

你将获得：① 一条从 YAML 的 fsdp_modules: [model.layers.{*}, model.embed_tokens, lm_head] 三行声明、经 _build_parallel_config 转成 FSDPPlanConfig.apply_modules 字典、再由 MindSpeedParallelEngine.apply_fsdp_modules 喂给 fully_shard_parallel_modules 对每个匹配模块原位分片的完整落地链路 ② 弄懂 {*} 通配如何把"层路径模式"展开到 Qwen3-8B 全部 transformer 层、以及 fsdp_size == world_size 这条硬约束的出处与含义——并看清 embed_tokens / lm_head 是否分片是可以单独声明的、并非"全模型一刀切"

你已经熟悉的情况 桥接开场

你训练过单卡模型的话，对"参数在哪儿"这件事大概没有疑虑：model = AutoModelForCausalLM.from_pretrained(path) 一调，整个模型的几十亿参数全整整齐齐躺在这一张卡（CPU 或 GPU）的显存里，model.parameters() 里的每个 nn.Parameter 都是一整块 torch.Tensor，前向 backward 都在同一张卡上算。多卡训练要切，你用 DDP——但 DDP 的切法是"每张卡都持有一份完整模型副本，反向时 all-reduce 梯度"，参数本身没有真正切开，4 张卡上还是 4 份完整模型。显存瓶颈来了：Qwen3-8B bf16 全量约 16 GB 单卡放得下，但 optimizer state（Adam 是 2 倍参数量）+ 激活 + 梯度一加，单卡训不动——DDP 这时帮不上忙，因为它根本不省模型显存。

FSDP2 的解法是把"整张卡持整模型"换成"每张卡只持参数的一片"——per-parameter sharding。PyTorch 2.3+ 的 torch.distributed.fsdp.fully_shard(module) 是这个能力的原语：对一个 nn.Module 原位（in-place）改造，把它名下所有参数 shard 到一张 DeviceMesh 上，每卡只留自己那 1/N 片（变成 DTensor），前向自动 all-gather 临时拼回完整参数算完即弃、反向 reduce_scatter 把梯度也切成片。EP03 讲过这个概念，EP07 画过 DeviceMesh 与 engine 的六步 apply。本篇回答最后一个落地问题：这份"对哪些模块开分片"的清单从哪来、怎么匹配到模型上、谁真正调 fully_shard。答案在三个地方——YAML 的 parallel.fsdp_modules 列表、model_factory 把它装成 FSDPPlanConfig.apply_modules、engine 的 apply_fsdp_modules 拿它调 fully_shard_parallel_modules。这就是 config → 真实分片的那一跳。

本项目怎么做 原理

FSDP2 落地的核心模型是"声明式清单 + 原位匹配 + 原语包装"三步。声明式清单：用户在 YAML 写一个模块路径列表（带 {*} 通配），不写循环、不写 mesh、不写 rank，只声明"这些模块要分片"。原位匹配：分片函数拿到这份清单后，对模型的每个 named_module() 路径做通配匹配——model.layers.{*} 展开成 model.layers.0、model.layers.1… 直到 model.layers.N-1（Qwen3-8B 层数以 config 为准，详见下文），每个被命中的 Qwen3DecoderLayer 实例都收进待分片队列。原语包装：对队列里每个模块实例调一次 fully_shard(module, mesh=..., mp_policy=..., reshard_after_forward=...)，原位把它名下参数切成 DTensor 片，挂在 mesh 的 fsdp 轴上。三步串起来：YAML 三行 → 几十个 fully_shard 调用 → 每卡持 1/8 参数。下面的 SVG 画清了"模式 → 展开 → 每层分片"这一跳。

这张图的三个关键：① 声明只有 3 行，但落地是几十次 fully_shard——{*} 是放大器，把一行声明展开成 N 次调用；② 分片是 per-module 原位，不是"整模型先 clone 再切"——每个命中的 Qwen3DecoderLayer 被独立 fully_shard，层与层之间各自分片、各自的 all-gather/reduce_scatter 通信彼此独立（这也意味着每层前向都触发一次 all-gather，通信次数 = 层数，所以才有 prefetch 优化）；③ embed_tokens/lm_head 与 layers 是平级清单项，分片与否完全由是否出现在 fsdp_modules 列表里决定——不像 DDP 那样"要么全包要么不包"，这里可逐模块声明。

源码下钻 file:line

model_factory._build_parallel_config：把 fsdp_modules 列表转成 FSDPPlanConfig.apply_modules 字典
@staticmethod
def _build_parallel_config(model_args, parallel_args, dp_size) -> 'ParallelEngineConfig':
    """Builds the Config based on parallel_args and hardcoded layer name rules.
    Note: The wildcards here (e.g., 'model.layers.{*}') are suitable for standard
    structures like Llama/Qwen. If using other non-standard models, these strings
    might need adjustment.
    """
    # --- 1. FSDP Plan ---
    # Requirement: Apply FSDP to transformer layers
    apply_modules = {
        parallel_args.fsdp_modules[0]: {
            'reshard_after_forward': parallel_args.reshard_after_forward,
            'shard_placement_fn': parallel_args.shard_placement_fn,
        },
    }
    for modules in parallel_args.fsdp_modules[1:]:
        apply_modules[modules] = {
            'reshard_after_forward': parallel_args.reshard_after_forward,
        }
    fsdp_plan = FSDPPlanConfig(
        ignored_modules=parallel_args.ignored_modules if parallel_args.ignored_modules else [],
        apply_modules=apply_modules,
        param_dtype=parallel_args.param_dtype,
        reduce_dtype=parallel_args.reduce_dtype,
        num_to_forward_prefetch=parallel_args.num_to_forward_prefetch,
        num_to_backward_prefetch=parallel_args.num_to_backward_prefetch,
    )
见 ../mindspeed_llm/fsdp2/models/model_factory.py：:144（def _build_parallel_config，静态方法）、:148（docstring 注明 wildcards 适配 Llama/Qwen 标准结构，非标模型需手改）、:153（apply_modules 字典，首项 fsdp_modules[0] 带 reshard_after_forward + shard_placement_fn 两键）、:159（for fsdp_modules[1:] 循环，其余项只带 reshard_after_forward）、:163（FSDPPlanConfig 装配，apply_modules=apply_modules）、:120（fsdp_size = parallel_args.fsdp_size 取值）、:122（dp_size = world_size // (tp*fsdp*cp)）、:211（ParallelEngineConfig 装配，fully_shard_parallel_size=fsdp_size）

这段是 config → plan 的核心。两个要点：① 列表 → 字典：fsdp_modules（list）被转成 apply_modules（dict），key 是模块路径模式字符串（原样保留 {*}，这里不展开），value 是该模块的分片参数。② 首项特权：只有 fsdp_modules[0]（即 model.layers.{*}）带 shard_placement_fn，其余项（embed_tokens / lm_head）只带 reshard_after_forward——这是个隐含约定：列表第一项被视为"主体分片目标"，享有自定义 shard 放置函数的特权，其余项沿用默认放置。docstring（行 148）也直白提醒：{*} 这种通配"适用于 Llama/Qwen 的标准结构"，非标模型要手改路径字符串——说明 model.layers.{*} 是为这类"层堆叠"模型量身定的约定，不是任意 AST 解析。展开 {*} 这一步不在此函数内，apply_modules 的 key 原样带着 {*} 传给下游，真正的展开发生在分片函数对 model.named_modules() 做匹配时（核实：{*} 通配展开实现位于 fsdp_turbo.distributed.fully_shard_parallel.fully_shard_parallel_modules 内部，非本仓源码，最终以该包源码为准）。

MindSpeedParallelEngine.apply_fsdp_modules：把 plan 喂给 fully_shard_parallel_modules
class MindSpeedParallelEngine(torch.nn.Module):
    def __init__(self, config, model, init_device="cpu", weights_path=None, hf_config=None):
        super().__init__()
        self.config = config
        self.model = model
        self.init_device = init_device
        self.weights_path = weights_path
        self.hf_config = hf_config
        self.parallel_state = init_parallel_state(self.config)
        self.apply_quantization_modules()
        self.apply_tp_modules()
        self.apply_ep_modules()
        self.apply_cp_modules()
        self.apply_recompute_modules()
        self.apply_fsdp_modules()          # FSDP 最后包，最外层
        if self.init_device == "meta":
            logger.info_rank0("> Loading weights after FSDP wrapping...")
            WeightLoader.load(model=self.model, weights_path=self.weights_path,
                              device=None, hf_config=self.hf_config)

    def apply_fsdp_modules(self):
        self.model = fully_shard_parallel_modules(
            self.model, self.parallel_state.get_fsdp_device_mesh(), self.config.fsdp_plan)

专家路径 唯一本仓内可见的 fully_shard 原语调用（见下文件），作 dense 路径的旁证
本仓唯一可见 fully_shard 原语调用处：专家分片函数直接调 torch.distributed.fsdp.fully_shard
import torch
from torch.distributed.fsdp import fully_shard, MixedPrecisionPolicy
from torch.distributed.tensor import Shard
from mindspeed.fsdp.utils.str_match import module_name_match   # 通配匹配原语（外部 mindspeed 包）

def expert_fully_shard_modules(model, efsdp_mesh, plan):
    efsdp_modules = get_efsdp_modules(model, plan)
    config = {'mesh': efsdp_mesh,
              'mp_policy': MixedPrecisionPolicy(param_dtype=torch.bfloat16,
                                                reduce_dtype=torch.float32),
              'shard_placement_fn': get_shard_placement_fn()}
    for experts in efsdp_modules:
        if isinstance(experts, torch.nn.ModuleList):
            for expert in experts:
                fully_shard(expert, **config)            # 原位分片原语
                set_gradient_divide_factor(expert, plan._gradient_divide_factor)
        else:
            fully_shard(experts, **config)
            set_gradient_divide_factor(experts, plan._gradient_divide_factor)
    return model

def get_efsdp_modules(modules, plan):
    efsdp_modules = []
    for plan_name in plan.apply_efsdp_modules:           # 如 model.layers.{*}.mlp.experts
        for name, module in modules.named_modules():
            if module_name_match(plan_name, name):       # 通配匹配在这里发生
                efsdp_modules.append(module)
    if len(efsdp_modules) == 0:
        raise RuntimeError(...)
    return efsdp_modules

一个微妙但重要的差异：dense 路径的 apply_modules 是 dict（key=路径模式，value=分片参数），专家路径的 apply_efsdp_modules 是 list（只有路径模式，分片参数在函数内硬编码 MixedPrecisionPolicy(bf16/fp32)）。这意味着 dense 模型可以逐模块声明不同的 shard_placement_fn（model_factory 行 153 首项特权），专家模型则所有 expert 共用一套 placement。这是 dense 与 MoE 在分片粒度上的设计差异，EP09 会展开。

这就是"fsdp_size 须等于 world_size"的硬出处。行 91 的判定直接 fully_shard_parallel_size != torch.distributed.get_world_size() 就抛——没有 fallback、没有自动补齐。把它和 model_factory 行 122 的 dp_size = world_size // (tp*fsdp*cp)、行 88 的 dp != 1 抛合起来看，三条约束联立推出 Qwen3-8B 在 A2 场景的唯一合法解：tp_size=1, cp_size=1, fsdp_size=8=world_size（A2 脚本正是这么传的，行 23-24）。fsdp_size 必须=world_size 意味着FSDP 是"全卡参与的单一分片组"，不存在"只在一部分卡之间分片"的中间档——这把 DDP 那种"4 卡一组、组间再 dp"的灵活性砍掉了，换来的是 per-parameter 切到极致的显存省。这是当前 fsdp2 后端的有意限制，不是 bug（核实：措辞 "Currently" 表明是阶段性约束，后续版本或放宽，以源码为准）。

GPU / 已知（DDP）

参数持有每卡一份完整副本，4 卡=4 份整模型
切分对象不切参数，只 all-reduce 梯度
显存省不省模型显存，只省 optimizer 通信
粒度整模型一刀切，DDP(model) 包或不包
通信反向一次 all-reduce 梯度
embed/lm_head与主体同进同退，无单独控制

NPU / 本项目（FSDP2）

参数持有每卡持 1/world_size 片 DTensor，全卡合一份
切分对象per-parameter shard，参数本身切开
显存省省模型+optimizer+梯度，单卡训得动大模型
粒度fsdp_modules 声明式，逐模块可单独控
通信每层前向 all-gather + 反向 reduce_scatter
embed/lm_head可单独声明分片；tie 时需配合（model_loader._post_process）
小结　心智模型：fsdp_modules 是一份"对哪些模块开分片"的声明式清单——它本身不切参数、不碰 mesh、不算 rank，只是几行字符串；真正的展开（{*} → 每一层）、真正的切（fully_shard 原语）由 fully_shard_parallel_modules 在拿到 plan+mesh+model 后完成。从 YAML 到显存省下来，中间只过了两个函数：model_factory 把 list 装成 FSDPPlanConfig.apply_modules 字典，engine 把字典连同 fsdp DeviceMesh 喂给分片原语。fsdp_size == world_size 是这套路成立的硬前提——它把"FSDP 全卡一组"焊死，换来 per-parameter 切到 1/N。
常见误区：① 以为 fsdp_size 可任意小于 world_size——不行，parallel_state.py:91 直接抛 must be equal to the distributed world size，当前后端只支持 fsdp 组=全卡这一档（"Currently" 措辞暗示是阶段性限制）。② 以为 FSDP 是"全模型一刀切"——不是，fsdp_modules 是逐模块声明的：layers、embed_tokens、lm_head 各自一行，是否分片完全看是否进列表；embed_tokens/lm_head 可单独控，甚至能只切 layers 不切头（核实：取舍由配置决定，模型_loader._post_process 还会处理 tie_word_embeddings 下 embed 与 lm_head 共享 weight 的分片语义）。③ 以为 {*} 在 model_factory 里就展开了——没有，_build_parallel_config 只把 {*} 原样塞进 dict 的 key，展开发生在下游 fully_shard_parallel_modules（dense）/ module_name_match（expert）对 named_modules() 的匹配里，这两处源码都不在本仓（分别属 fsdp_turbo 与 mindspeed 外部包）。④ 以为列表第一项与后续项等价——不等价，首项（model.layers.{*}）独享 shard_placement_fn 特权（model_factory:153），后续项只带 reshard_after_forward，改 shard 放置策略只对主体生效。
