class Qwen3RMSNorm(nn.Module):
    def forward(self, hidden_states):
        input_dtype = hidden_states.dtype
        hidden_states = hidden_states.to(torch.float32)
        variance = hidden_states.pow(2).mean(-1, keepdim=True)
        hidden_states = hidden_states * torch.rsqrt(variance + self.variance_epsilon)
        return self.weight * hidden_states.to(input_dtype)

def apply_rotary_pos_emb(q, k, cos, sin, position_ids=None, unsqueeze_dim=1):
    cos = cos.unsqueeze(unsqueeze_dim)
    sin = sin.unsqueeze(unsqueeze_dim)
    q_embed = (q * cos) + (rotate_half(q) * sin)
    k_embed = (k * cos) + (rotate_half(k) * sin)
    return q_embed, k_embed
差异讲解

左侧为 HF 上游 Qwen3RMSNorm.forward 与 apply_rotary_pos_emb 的标准 GPU 实现：RMSNorm 用 Python-level 的 pow(2).mean().rsqrt() 三步 + 末尾 upcast/downcast，多次中间张量；RoPE 用 (q*cos)+(rotate_half(q)*sin) 的逐元素乘加，rotate_half 还要做 concat-flip。

右侧 patch 后：npu_rms_norm 把整个 RMSNorm 融成一个 NPU CANN 算子（一次 kernel launch、无中间 fp32 物化、权重乘法内置），npu_rotary_mul 把 cos/sin 乘加 + rotate_half 融成单个算子。差异本质：不是数值改动（数学等价），而是 kernel 融合省访存、省 launch；DDP/GPU 侧若要在 N 卡上跑同模型做对照，走的是左侧标准实现（PyTorch eager 或 fused kernel via torch.compile），不会调 torch_npu。这也解释了为何 patch 受 use_fused_* 开关保护——关掉即回到左侧等价但更慢的路径，便于跨厂商数值对拍。

最后把本文件放回 FSDP2 + NPU 适配的大图，回答“为什么不能直接拷 HF”：

第一座桥（HF backbone 复用）：:29 整段 transformers.Qwen3Model(config)——主干算子不重写，靠 patch + attn_implementation 路由接 NPU kernel。这样 HF 升级 Qwen3 时本壳自动跟进。

第二座桥（DTensor-aware 改写）：:31 自定义 LMHead（modules.py 里 .to_local()）+ :48 loss_ctx 钩子。FSDP2 的 fully_shard 会把 lm_head.weight 包成 DTensor（Shard(0) colwise），forward 时必须 to_local() 取本 rank 分片再算；HF 原版 Qwen3LMHead 假设 weight 是普通 Tensor，直接 F.linear 会拿不到分片语义。这是为接 fully_shard 必须改写的点，不能照搬 HF。

第三座桥（NPU 融合算子）：:105-115 只 patch RMSNorm/RoPE 两个无 HF NPU 等价的算子，kernel 实现在 torch_npu（本仓 common/fusions.py 是薄 wrapper）。Attention 不 patch，走 config 的 attn_implementation 路由。

EP12 场景下（8 卡 NPU、8B 模型 FSDP2 全分片）三桥同时生效：fully_shard 切参数、LMHead.to_local() 取本分片算 logits、loss_ctx 省 [B,T,vocab] 显存、npu_rms_norm/npu_rotary_mul 省 backbone 访存。本文件是三桥的组装点，而非任一桥的实现细节。

        >>> model = Qwen3ForCausalLM.from_pretrained("Qwen/Qwen3-8B")
