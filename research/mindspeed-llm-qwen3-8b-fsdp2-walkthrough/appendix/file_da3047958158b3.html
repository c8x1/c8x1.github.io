差异讲解

在 HuggingFace 的 Qwen3 实现（或你在 A100/H100 上跑的 DDP 版本）里，lm_head 就是一个普通的 nn.Linear(hidden_size, vocab_size, bias=False)，weight 是普通 torch.Tensor，forward 直接 F.linear(hidden_states, self.weight)。这里不需要 import DTensor，DDP 下 weight 各 rank 各持一份完整副本（replicated），前向各算各的、反向再 allreduce 梯度。

差异的核心：DDP 是“权重 replicated、梯度 allreduce”；FSDP2 是“权重被 fully_shard 切成 DTensor、各 rank 只持有本地分片、前向/反向按需 all-gather”。所以 NVIDIA DDP 路径里 self.weight 永远是普通 tensor，本文件第 10 行那个 isinstance(self.weight, DTensor) 分支在 DDP 下根本走不到——它是 FSDP2 专用的兼容层。

行 7-8 · 类定义与 forward 签名（含 loss_ctx 可选回调）

class Qwen3LMHead(nn.Linear):
    def forward(self, hidden_states: torch.Tensor, loss_ctx: callable = None):

第 7 行 class Qwen3LMHead(nn.Linear): 继承自 nn.Linear，所以它复用了 __init__（创建 weight 形状 [out_features, in_features]、可选 bias），只重写了 forward。第 8 行签名 forward(self, hidden_states, loss_ctx=None) 比标准 nn.Linear 多了一个 loss_ctx: callable = None 参数——这正是 FSDP2/Qwen3 训练路径的关键扩展点。

loss_ctx 是一个可调用对象，用来替代“先算完整 logits 再算交叉熵”的朴素路径。在 EP12（专家并行 12 卡 / 大词表）场景下，vocab_size 很大（Qwen3 词表约 15万+），若先 materialize 出 [batch*seq, vocab] 的 logits 再算 loss，显存会爆。所以调用方可以传一个 loss_ctx，把“logits 计算 + loss 计算”融合成 chunked/分块形式，只返回 loss、不返回完整 logits。本文件不强求 loss_ctx 内部实现，只做分派。

行 9-23 · DTensor 权重/偏置的本地化降级（to_local）

        # Handle distributed tensor (DTensor) weights and biases by converting to local tensors.
        if isinstance(self.weight, DTensor):

            if self.bias is not None:
                if not isinstance(self.bias, DTensor):

                        f"but got bias of type {type(self.bias)}."

                b = self.bias.to_local()
            else:
                b = None
        else:
            w = self.weight
            b = self.bias

这是本文件技术含量最高的一段。第 10 行 if isinstance(self.weight, DTensor): 探测 weight 是否已被 fully_shard 分片。若是，第 11 行 w = self.weight.to_local() 取出本 rank 持有的本地物理分片——把“逻辑 DTensor”降级成普通 torch.Tensor。注意这是 view/浅取，不是完整 all-gather：to_local() 只返回当前 rank 已有的那一片，不跨 rank 通信。

第 12-20 行处理 bias 的对称性：如果 weight 是 DTensor 而 bias 存在，bias 也必须是 DTensor（否则第 14-17 行抛 TypeError——这是一种前置一致性校验，防止“weight 分片了、bias 没分片”这种会让 F.linear 形状对不上的隐患静默通过）。第 19-20 行：无 bias 时 b = None（Qwen3 的 lm_head 用 bias=False，走的就是这条）。第 21-23 行 else 分支：weight 不是 DTensor（DDP 或未分片场景）时，直接用原始 self.weight/self.bias。

为什么必须 to_local()？因为下面第 27 行用的是 F.linear(hidden_states, w, b)——这是 eager 级算子，不认识 DTensor 的 placements。FSDP2 的语义是：fully_shard 的模块在前向会先 all-gather 出完整 weight（由 FSDP 的 hook 注入），然后 weight 在那一瞬是 DTensor 但本地已持有全量数据；to_local() 就是把这个“本地全量视图”剥出来喂给普通算子。这就是“DTensor-aware LMHead”的含义：它知道 weight 可能是 DTensor，主动降级，而不是假设 weight 永远是普通 tensor。

行 25-28 · 无 loss_ctx 分支：朴素前向返回 logits

        if loss_ctx is None:
            # If no loss context is provided, compute and return logits normally.
            logits = F.linear(hidden_states, w, b)
            return logits, None

第 25-28 行是 loss_ctx is None 的朴素路径：直接 logits = F.linear(hidden_states, w, b) 并 return logits, None。注意返回的是二元组 (logits, None)——即使不算 loss 也要返回一个 None 占位，保证调用方解包签名一致（见 qwen3.py:89： logits, loss = self.lm_head(..., loss_ctx=loss_ctx)）。这种“固定二元组返回 + 用 None 填充不用的槽”是 FSDP2 训练框架里 lm_head 的通用契约，调用方据此决定走推理（用 logits）还是训练（用 loss）。

在 EP12 大词表训练里，这条分支通常不会被走——训练时调用方会传 loss_ctx 走第 32 行的融合路径以省显存。推理时（只要 logits、不要 loss）才走这条。

行 29-33 · 有 loss_ctx 分支：融合 loss 与死代码诚实声明

        else:
            # Otherwise, delegate loss computation to the provided loss context function,
            # which typically enables memory-efficient or chunked loss calculation.
            return None, loss_ctx(hidden_states, w, b)

第 29-32 行：return None, loss_ctx(hidden_states, w, b)。把 hidden_states 和已本地化的 w/b 一起交给 loss_ctx，由它内部决定如何分块算 logits+交叉熵，最终只返回 loss（logits 槽填 None）。这就是前文说的“chunked/内存高效 loss 计算”的落地点——本文件不实现 loss_ctx 的细节，只做委托。结合第 11 行的 to_local()，loss_ctx 拿到的 w 是本地物理分片，可以在分块维度上做局部计算再归约。

诚实声明（死代码）：这个类 Qwen3LMHead 实际上从未被引用。全仓库 grep Qwen3LMHead 仅命中本文件第 7 行的定义处，没有任何 import 或实例化。dense 模型 qwen3.py:18,31 和 MoE 模型 qwen3_moe.py:34,48 都是从 mindspeed_llm.fsdp2.models.common.modules 导入 LMHead，而 common/modules.py 里的 class LMHead 与本文件 Qwen3LMHead 逐字节相同（仅类名不同，forward 逻辑、DTensor 处理、loss_ctx 分派完全一致）。所以：真正在 EP12 训练里承担 lm_head 角色的是 common.modules.LMHead，本文件是一份未被接入的副本。读这段代码的价值在于理解“DTensor-aware LMHead”的设计模式本身（to_local 降级 + loss_ctx 委托 + 二元组返回契约），这一模式在 common.modules.LMHead 里才是活代码、在 Qwen3LMHead 里是死代码。维护时若要改 lm_head 行为，应改 common 版，本文件可清理删除。

import torch
import torch.nn.functional as F
from torch import nn
from torch.distributed.tensor import DTensor


class Qwen3LMHead(nn.Linear):
    def forward(self, hidden_states: torch.Tensor, loss_ctx: callable = None):
        # Handle distributed tensor (DTensor) weights and biases by converting to local tensors.
        if isinstance(self.weight, DTensor):
            w = self.weight.to_local()
            if self.bias is not None:
                if not isinstance(self.bias, DTensor):
                    raise TypeError(
                        f"Expected bias to be a DTensor when weight is a DTensor, "
                        f"but got bias of type {type(self.bias)}."
                    )
                b = self.bias.to_local()
            else:
                b = None
        else:
            w = self.weight
            b = self.bias

        if loss_ctx is None:
            # If no loss context is provided, compute and return logits normally.
            logits = F.linear(hidden_states, w, b)
            return logits, None
        else:
            # Otherwise, delegate loss computation to the provided loss context function,
            # which typically enables memory-efficient or chunked loss calculation.
            return None, loss_ctx(hidden_states, w, b)
