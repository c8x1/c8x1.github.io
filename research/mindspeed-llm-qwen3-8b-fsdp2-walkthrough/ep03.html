并行范式：DDP → FSDP1 → FSDP2

目标：把 AllReduce 拆成 ReduceScatter + AllGather。前置：EP02 硬件与运行时。

你将获得：① 一张 DDP AllReduce 与 FSDP2 RS+AG 的对照小图，看清"一次 AllReduce 被拆成两半"的物理含义 ② FSDP2 参数的 5 态生命周期（Fully Sharded → AllGather → Compute → ReduceScatter → Update）+ DTensor/DeviceMesh/MixedPrecisionPolicy/Prefetch 四块基石的定位

你已经熟悉的情况 桥接开场

在 DDP（DistributedDataParallel）的世界里，每张卡都端着一个完整的模型副本，各吃一个独立的数据 Batch，前向各算各的；反向算完梯度后，所有卡通过一次 AllReduce 把梯度同步成一致，再各自就地更新。心智很简洁：每卡全量副本 + 梯度 AllReduce，显存账单里"参数 / 梯度 / 优化器状态"三项都按"完整模型大小 ×1"计。模型一大，单卡 HBM（EP02 已见 NPU 侧 64GB 硬约束）就装不下了。

FSDP（Fully Sharded Data Parallel）就是来填这个坑的：既然三项显存大头都"每卡一份完整副本"，那就把它们切片（Sharding），每张卡只存 1/N——用一次额外的通信，把"装不下"换成了"装得下"。本篇要讲清楚的核心一句话：把 DDP 那一次 AllReduce，拆成 ReduceScatter + AllGather 两段。这就是从 DDP 跨到 FSDP2 的过桥点。

本项目怎么做 原理

先看"通信被拆成两半"长什么样。下图左列是你熟的 DDP：反向算完整梯度 → 一次 AllReduce（每张卡拿到完整且一致的梯度）→ 各自更新完整参数。右列是 FSDP2：反向算出的完整梯度立刻 ReduceScatter（归约并切片，每卡只留 1/N 的梯度片）→ 优化器只用切片梯度更新切片参数 → 下次前向前再 AllGather 把切片参数拼回完整参数用于计算。AllReduce = ReduceScatter + AllGather 的和，FSDP2 只是把它掰开，并在两段之间塞进了"切片存储 + 切片更新"。

图 3.1　DDP AllReduce vs FSDP2 ReduceScatter+AllGather：DDP 用一次 AllReduce 让每卡拿到完整梯度、保留全量参数；FSDP2 把这次 AllReduce 拆成 ReduceScatter（出切片梯度、更新切片参数）与 AllGather（用前拼回完整参数）两段，参数/梯度/优化器状态在静止态只占 1/N。

拆完通信，参数本身也得被切。FSDP2 把一个参数的生命周期分成 5 个状态，循环往复：

图 3.2　FSDP2 参数 5 态生命周期：Fully Sharded（静止·1/N）→ AllGather（拼回）→ Compute（用完整参数）→ ReduceScatter（归约切片）→ Update（切片更新）→ 回到 Fully Sharded。①②④ 是显存省下来的关键（静止态只占 1/N），③ 是计算态。

这五态不是凭空设计，是"通信被拆成两段"的直接后果：既然 AllGather 只在前向前拼回、ReduceScatter 只在反向内切片，那么参数就必然经历"切片静止 → 拼回 → 计算 → 切片归约 → 切片更新 → 再切片静止"的循环。剩下四块基石各管一件：DTensor 管"切片后还能当成完整 Tensor 用"的逻辑/物理分离，DeviceMesh 管"切到哪些设备上"，MixedPrecisionPolicy 管"存什么精度、算什么精度、通信什么精度"，Prefetch 管"通信和计算怎么掩盖起来"。

① FSDP1 vs FSDP2——从 FlatParameter 到 Per-Parameter Sharding

FSDP1（Legacy）：将层内的所有参数拉平拼接，切分这个巨大的 1D 向量。破坏了原始模型参数结构。
FSDP2（New）：保持模型原始参数结构不变。每个参数（nn.Parameter）被单独切分并管理。
  → 极高组合性，可与 Tensor Parallel (TP) 或 Checkpointing 结合

docs/zh/pytorch/features/fsdp2/fsdp2_basic_features.md:9（FSDP1 详见 :11，FSDP2 详见 :12）

这一段把 FSDP1/FSDP2 的分水岭钉死：FSDP1 把参数打平成一个 FlatParameter 1D 向量再切，破坏了原始参数结构，自定义初始化、特定层微调都难做；FSDP2 逐参数（Per-Parameter）切分，结构不变，组合性高（与 TP/Checkpointing 可叠加）。本系列后续走读的 mindspeed_llm/fsdp2/ 走的是 FSDP2 这一支。注意：FSDP2 通过 torch.distributed.fsdp.fully_shard 做原位（In-place）并行化，不再像 FSDP1 那样用 Python 类包装模型（见文档 :3）。

② 工作原理——把 DDP 的 AllReduce 分解为 Reduce-Scatter + All-Gather

DDP：每个 Rank 拥有完整模型副本，处理独立 Batch，最后 All-Reduce 同步梯度。
FSDP：对参数、梯度、优化器状态进行切片(Sharding)，显著降低显存占用。
FSDP 将 DDP 的 All-Reduce 操作分解为 Reduce-Scatter 和 All-Gather。

docs/zh/pytorch/features/fsdp2/fsdp2_basic_features.md:16（DDP 详见 :16，FSDP 切片 :18，分解句 :22）

这就是图 3.1 的文字来源："FSDP 将 DDP 的 All-Reduce 操作分解为 Reduce-Scatter 和 All-Gather"（:22）——本篇标题"把 AllReduce 拆成 ReduceScatter + AllGather"即出自此句。DDP 每卡全量副本 + AllReduce（:16）；FSDP 通过切片参数/梯度/优化器状态降显存（:18）。一句关键认知：FSDP2 里已没有 AllReduce 这一通信原语，取而代之的是 RS（反向内）+ AG（前向前）。

③ FSDP 参数生命周期 5 态——静止/准备/计算/同步/更新

1. Fully Sharded（静止态）：Forward/Backward 之外，参数完全切分（每卡只存 1/N）
2. AllGather（准备态）：Forward/Backward 开始前，切片参数被聚合为完整参数
3. Compute（计算态）：使用完整参数进行计算
4. ReduceScatter（同步态）：Backward 内，完整梯度被归约并切分为切片梯度
5. Update（更新态）：优化器用切片梯度更新切片参数，优化器状态也是切分的

docs/zh/pytorch/features/fsdp2/fsdp2_basic_features.md:28（:28 :29 :30 :31 :32 五态逐条）

这就是图 3.2 的文字来源。五态逐条对应文档 :28–:32：① Fully Sharded 每卡 1/N（:28）；② AllGather 前向/反向开始前拼回（:29）；③ Compute 用完整参数算（:30）；④ ReduceScatter 反向内归约切片（:31）；⑤ Update 切片梯度更新切片参数、优化器状态也切分（:32）。注意第 5 态的附带效果：优化器状态天然也是切分的——这也是 FSDP 省显存的大头之一（DDP 里 AdamW 的两份状态量是每卡全量）。

④ DTensor——逻辑视图与物理视图分离（底层基石）

逻辑上：参数看起来仍是完整 Tensor（如 [4096, 4096]），保持单卡一致的编程体验
物理上：参数被切分并分布在 DeviceMesh 定义的设备组中（每卡只持 [512, 4096] Local Tensor）
DeviceMesh：描述设备拓扑，天然支持多维并行（2D FSDP，或 FSDP + TP）

docs/zh/pytorch/features/fsdp2/fsdp2_basic_features.md:40（逻辑视图 :40，物理视图 :42，DeviceMesh :44）

DTensor（torch.distributed.tensor.DTensor，见 :36）是 FSDP2 的底层基石。它的核心是逻辑/物理分离：你的代码仍把它当完整 Tensor [4096, 4096] 操作（:40），物理上却已切片分布在 DeviceMesh 定义的设备组里，每卡只有 [512, 4096] 的 Local Tensor（:42）。DeviceMesh（:44）描述拓扑，使得 FSDP2 天然支持多维并行——同一张 Mesh 换个维度就是 TP，叠加就是 FSDP+TP（这呼应了 :12 的"组合性"）。这是 EP07 并行状态引擎、EP08 分片落地要深入的地方。

⑤ MixedPrecisionPolicy——存储/计算/通信三精度分离（代码示例）

import torch
from torch.distributed.fsdp import MixedPrecisionPolicy

# FSDP2 混合精度转换流程
mp_policy = MixedPrecisionPolicy(param_dtype=torch.bfloat16, reduce_dtype=torch.float32)
# Forward: Parameters (FP32 storage) -> Cast to BF16 -> Compute
# Backward: Gradients (BF16) -> Cast to FP32 -> AllReduce

docs/zh/pytorch/features/fsdp2/fsdp2_basic_features.md:59（MixedPrecisionPolicy 三精度释义见 :48–:52）

MixedPrecisionPolicy（:48）把存储精度 / 计算精度 / 通信精度三者分开拧：param_dtype 管计算——前向/反向前把参数 cast 成 bfloat16（:50）；reduce_dtype 管通信——梯度同步（ReduceScatter）阶段 cast 成 float32 保数值稳定（:51）；buffer_dtype 独立管 Buffer（如 BatchNorm 统计量）精度防溢出（:52）。代码示例（:59）正是 MixedPrecisionPolicy(param_dtype=torch.bfloat16, reduce_dtype=torch.float32)。注意 :61 注释仍写 AllReduce——这是上游文档措辞遗留，按本篇图 3.1，FSDP2 实际走的是 RS（reduce_dtype 生效处）+ AG，已无 AllReduce 原语。

⑥ 通信与计算掩盖——Prefetching

为实现极致训练效率，FSDP2 实现了高度优化的通信计算掩盖(Overlap)机制，即 Prefetching。
（图：prefetch.png —— 在计算当前层的同时，预取下一层的参数切片）

docs/zh/pytorch/features/fsdp2/fsdp2_basic_features.md:66（prefetch 配图见 :68–:70）

Prefetch（:66）是 FSDP2 把通信塞进计算间隙的关键机制：在计算当前层的同时，提前发起下一层的 AllGather，让通信时间被计算时间"盖住"。这正是图 3.1 右列"② AllGather"和图 3.2 第 ② 态能不拖慢训练的原因。EP10 通信掩盖会专门下钻到 NPU 侧 HCCL + 多流 + Prefetch 的实现细节，本篇只立概念。另注文档 :72：本文示例基于 PyTorch 2.7 及以上版本；FSDP2 自 PyTorch 2.4 起以技术预览形式引入——与 EP02 钉死的镜像基线 PyTorch 2.7.1（docker/OVERVIEW.zh.md:200）对齐。

把六段叠起来：② 拆 AllReduce 为 RS+AG 是本篇的"过桥点"，③ 五态是被拆后参数不得不走的循环，④ DTensor/DeviceMesh 给切片提供了"仍当完整 Tensor 用"的抽象与多维拓扑，⑤ MixedPrecisionPolicy 把存/算/通信三精度分开拧，⑥ Prefetch 把通信塞进计算间隙。这五块基石共同回答了"FSDP2 怎么用通信换显存、又不被通信拖死"。真实代码（mindspeed_llm/fsdp2/）留到 EP07–EP11。

小结　心智模型：FSDP2 ＝ DDP 的"参数也切片"版——用一次额外的通信，把每卡全量副本换成 1/N 切片。核心是把 DDP 那一次 AllReduce 拆成 ReduceScatter（反向内、出切片梯度）+ AllGather（前向前、拼回完整参数）两段，于是参数不得不走 Fully Sharded → AllGather → Compute → ReduceScatter → Update 五态循环，静止态只占 1/N；DTensor/DeviceMesh 给切片提供了"逻辑完整、物理切片"的抽象与多维拓扑，MixedPrecisionPolicy 把存/算/通信三精度分开拧，Prefetch 把通信塞进计算间隙。一句话：FSDP2 用通信换显存，用 Prefetch 换回被吃掉的时间。
常见误区：(1) 以为 FSDP2 里还有 AllReduce——已无，梯度同步走的是 ReduceScatter + AllGather（basic_features.md:22），文档 :61 注释里的"AllReduce"是上游措辞遗留，别被它带回去；(2) 以为 fsdp_size 可以随便挑（比如设成 2、留一半卡做别的）——fsdp_size 须等于 world_size：FSDP 的切片粒度就是全部参与训练的卡，分片数 N＝world_size，每卡存 1/N；若想"一部分卡切片、一部分卡干别的"，那不是调 fsdp_size，而是上 2D Mesh（FSDP+TP，见 :44）这种多维并行（示意，最终以源码为准），EP07/EP08 再下钻。
