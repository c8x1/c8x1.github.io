Checkpoint 与权重转换

目标：讲清 dcp 分片 ckpt 与 HF 中转。前置：EP16 优化器与调度。

你将获得：① 三类文件的逐段精读——checkpoint_manager.py 如何用 torch.distributed.checkpoint 把分片模型/优化器状态落盘与 resume（save/load/execute_save 三入口）、weight_conv_adapter.py 如何在加载时把 HF 键名按 transformers 的 conversion_mapping 改写成模型侧键、merge_dcp_to_hf.py 如何只读 DCP metadata 分片合并出 HF safetensors，每段贴真实 file:line ② 一个反直觉点——FSDP2 的 dcp 分片 ckpt 不能被 transformers 直接 from_pretrained 加载：dcp 存的是每 rank 的本地分片 + DTensor 布局元信息，必须经 merge_dcp_to_hf 把分片重新拼回全量再写成 HF safetensors 才能用；而 HF 这个"全量 safetensors"格式同时是训练前的入料和训练后的产物，是 NPU↔GPU、训练↔推理之间的中立中转格式。

你已经熟悉的情况 桥接开场

你写过 torch.save(model.state_dict(), "model.pt") / model.load_state_dict(torch.load("model.pt"))。单卡世界里 state_dict() 返回一个 OrderedDict[str, torch.Tensor]——键是参数全名（model.layers.0.self_attn.q_proj.weight），值是全量张量；torch.save 把整个 dict pickle 到一个文件，torch.load 整份读回。HuggingFace 这边则是 model.save_pretrained("dir/") 写出 model.safetensors（或 model-0000X-of-0000Y.safetensors + model.safetensors.index.json），from_pretrained("dir/") 读回——safetensors 比 pickle 安全（无代码执行）、加载快、支持 mmap。这套全量 ckpt 的世界观你很熟：一个文件（或一组分片文件）= 整个模型的所有参数，每张卡都加载完整副本。

多卡 DDP 下你大概率还是用 torch.save——但通常只在 rank0 存一次 model.module.state_dict()（DDP 包的是副本，每卡都有完整模型），torch.save 出来仍是全量。问题来了：FSDP2 下没有任何一张卡持有完整模型。参数被 fully_shard 切成 N 片（EP08 讲过），每卡只有 1/N 的物理数据（DTensor 的 local shard），优化器状态 m/v 也只跟本地这片同形（EP16 讲过）。这时候 model.state_dict() 返回的是分片张量，torch.save 直接存会丢失"这张是全量的哪一片、布局是什么"的元信息，换一个并行度（比如 8 卡训完要迁到 4 卡推理）根本拼不回去。所以 FSDP2 不能再裸用 torch.save——它改用 PyTorch 官方的 Distributed Checkpoint（dcp），torch.distributed.checkpoint 这个torch 自带（非外部）模块负责把每卡分片按 rank 写成一组文件 + 一份 metadata，加载时按当前拓扑 reshape。

这一篇要厘清的关键认知是：dcp 是"训练期分片 ckpt"，HF safetensors 是"交换期全量 ckpt"——两者不是一回事，也不是同一层。训练里 save/resume 走 dcp（分片、跨拓扑可恢复），但训练完要把权重拿去推理（GPU 上的 transformers、或 vLLM、或别的卡数），就必须把 dcp 的分片合并回 HF 全量 safetensors——这就是 merge_dcp_to_hf.py 存在的理由。HF 在这里扮演中立中转格式：训练前从 HF 入料（weight_conv_adapter 改键名加载），训练后产出 HF（merge_dcp_to_hf 合并写出），跨硬件迁移（GPU ckpt 不能直用）也得绕 HF 中转。下面把这三类文件 + 根目录两个入口脚本 + 官方文档逐段拆开读。

本项目怎么做 原理

核心原理一句话：dcp 不存"全量权重"，存"每 rank 的本地分片 + 一份描述谁是谁的 metadata"。save 时，每个 rank 把自己负责的那 1/N 参数 + 1/N 优化器状态写成一个（或一组）文件，dcp 另写一份 metadata 记录每个张量键→(shape, dtype, 分片布局)；load 时，各 rank 读 metadata 知道"我要的这部分在哪些文件里"，按当前拓扑（rank 数、mesh 维度）reshape 后填进自己的本地分片。所以 dcp 的最大特性是拓扑无关——8 卡存的 ckpt 能在 4 卡上 resume（dcp 内部按当前 rank 数重新分片），这是裸 torch.save 做不到的。

HF 那边则是全量的 safetensors——一个（或几个）文件装下整份模型，键名是 HF 标准命名（model.layers.0.self_attn.q_proj.weight）。transformers.from_pretrained 直接 mmap 读、按需切。HF 全量 ckpt 的好处是通用——任何推理框架、任何卡数都能加载；坏处是大且不含训练状态（只有参数，没有优化器 m/v、step、lr）。所以训练内部 save/resume 不用 HF（太大、丢训练状态），训练产出给推理用才转 HF。merge_dcp_to_hf.py 做的就是"读 dcp metadata → 估算每张张量大小 → 按目标 shard_size 分组 → 逐组 load 分片 + full_tensor() 拼全量 → 写 safetensors"，全程只按需物化一个 shard，内存峰值 = 一个 shard 大小（不是整个模型），这是它能处理 8B+ 的关键。下图把整条权重流转摊开。

@classmethod
def load(
    cls,
    path: str,
    state: Dict[str, Any],
    process_group=None,
    storage_reader: Optional[FileSystemReader] = None,
) -> Dict[str, Any]:
    if state is None or "model" not in state:
        raise ValueError("State with model must be provided for loading.")

    checkpoint_dir = path
    load_state = {"model": ModelState(state["model"])}

    # Determine which components exist in checkpoint
    saved_keys = cls._get_saved_keys(checkpoint_dir)
    if "optimizer" in state and "optimizer" in saved_keys:
        load_state["optimizer"] = OptimizerState(
            model=state["model"],
            optimizer=state["optimizer"],
        )

    if storage_reader is None:
        storage_reader = cls._create_storage_reader(checkpoint_dir)

    # Perform DCP load
    dcp.load(
        state_dict=load_state,
        storage_reader=storage_reader,
        process_group=process_group,
    )

    # Load extra_state if present
    if "optimizer" in saved_keys:
        cls._load_extra_state(checkpoint_dir, state)

    logger.info_rank0(f"Loaded checkpoint from {checkpoint_dir}")
    return state

@classmethod
def execute_save(
    cls,
    save_state: Dict[str, Any],
    storage_writer: FileSystemWriter,
    save_async: bool,
) -> None:
    if cls._metadata_process_group is None:
        cls._metadata_process_group = dist.new_group(backend="gloo")
    if save_async:
        # Ensure previous async save is finished
        if cls.dcp_save_future is not None:
            cls.dcp_save_future.result()
            cls.dcp_save_future = None
            dist.barrier()

        cls.dcp_save_future = dcp.async_save(
            state_dict=save_state,
            storage_writer=storage_writer,
            process_group=cls._metadata_process_group,
        )
    else:
        dcp.save(
            state_dict=save_state,
            storage_writer=storage_writer,
            process_group=cls._metadata_process_group,
        )
        if dist.is_initialized():
            dist.barrier()

        # Explicit memory cleanup
        gc.collect()
        empty_cache()
        synchronize()

mindspeed_llm/fsdp2/checkpoint/checkpoint_manager.py —— ModelState / OptimizerState：Stateful 适配器；把 dcp 分片张量按 EP 维度 restore/drop

class ModelState(Stateful):
    @torch.no_grad()
    def state_dict(self):
        model_state_dict = get_model_state_dict(model=self.model)

        if self.should_ep_aware:
            if dist.get_rank() == 0:
                logger.info("ModelState: Restoring EP dimension for Expert modules")
            for name, tensor in model_state_dict.items():
                if name in self.ep_fqn2spec_info and torch.is_tensor(tensor) and tensor.ndim > 0:
                    model_state_dict[name] = restore_ep_dim(tensor, self.ep_fqn2spec_info[name].ep_fsdp_mesh)
        return model_state_dict

class OptimizerState(Stateful):
    @torch.no_grad()
    def state_dict(self):
        if self.should_ep_aware:
            # EP -> MultiOptimizer, which produces a merged flattened dict
            optim_sd = self.optimizer.state_dict()
            for name in list(optim_sd.keys()):
                ep_fqn = self._find_ep_fqn(name)
                if ep_fqn is None:
                    continue
                tensor = optim_sd[name]
                if torch.is_tensor(tensor) and tensor.ndim > 0:
                    optim_sd[name] = restore_ep_dim(tensor, self.ep_fqn2spec_info[ep_fqn].ep_fsdp_mesh)
            return optim_sd
        # Non-EP -> single optimizer
        return get_optimizer_state_dict(model=self.model, optimizers=self.optimizer)

def _get_sharding_plan(checkpoint_path, shard_size, save_dtype):
    reader = FileSystemReader(checkpoint_path)
    metadata = reader.read_metadata()
    if not isinstance(metadata, Metadata):
        raise ValueError(f"Invalid metadata format in {checkpoint_path}")

    tensor_infos = []
    all_dcp_keys = []
    for key, tensor_meta in metadata.state_dict_metadata.items():
        hf_key = _normalize_key(key)
        if hf_key is None:
            continue
        # Resolve dtype for size estimation
        if save_dtype:
            dtype = getattr(torch, save_dtype) if isinstance(save_dtype, str) else save_dtype
        else:
            if not hasattr(tensor_meta.properties, "dtype"):
                raise ValueError(f"Missing dtype info for tensor '{key}'")
            dtype = tensor_meta.properties.dtype
        numel = 1
        for dim in tensor_meta.size:
            numel *= dim
        byte_size = numel * get_dtype_size(dtype)
        tensor_infos.append({"dcp_key": key, "hf_key": hf_key, "size": byte_size, "metadata": tensor_meta})
        all_dcp_keys.append(key)
    # Deterministic ordering + greedy bin-packing into shards
    tensor_infos.sort(key=lambda x: x["hf_key"])
    shards = []; current_shard = {}; current_size = 0; total_size = 0
    for info in tensor_infos:
        size = info["size"]; total_size += size
        if current_shard and current_size + size > shard_size:
            shards.append(current_shard); current_shard = {}; current_size = 0
        current_shard[info["hf_key"]] = info["dcp_key"]; current_size += size
    if current_shard:
        shards.append(current_shard)
    return shards, total_size, all_dcp_keys

def _process_shard(shard_idx, num_shards, shard_keys, checkpoint_path, output_dir, save_dtype, safe_serialization):
    weights_name = SAFE_WEIGHTS_NAME if safe_serialization else WEIGHTS_NAME
    if num_shards == 1:
        filename = weights_name
    else:
        prefix, extension = weights_name.rsplit(".", 1)
        filename = f"{prefix}-{shard_idx + 1:05d}-of-{num_shards:05d}.{extension}"
    save_path = os.path.join(output_dir, filename)

    reader = FileSystemReader(checkpoint_path)
    metadata = reader.read_metadata()
    # Allocate placeholder tensors
    state_dict = OrderedDict()
    for dcp_key in shard_keys.values():
        tensor_meta = metadata.state_dict_metadata[dcp_key]
        state_dict[dcp_key] = torch.empty(tensor_meta.size, dtype=tensor_meta.properties.dtype)

    # Load tensors from DCP
    load(state_dict, checkpoint_id=checkpoint_path, storage_reader=reader, no_dist=True)

    processed_dict = OrderedDict()
    target_dtype = getattr(torch, save_dtype) if isinstance(save_dtype, str) else save_dtype
    for hf_key, dcp_key in shard_keys.items():
        tensor = state_dict[dcp_key]
        if hasattr(tensor, "full_tensor"):
            tensor = tensor.full_tensor()
        if target_dtype:
            tensor = tensor.to(dtype=target_dtype)
        processed_dict[hf_key] = tensor.cpu().detach().clone()
        del tensor
    # Memory cleanup
    del state_dict, metadata, reader
    gc.collect()
    if torch.cuda.is_available():
        torch.cuda.empty_cache()
    # Save shard
    if safe_serialization:
        save_file(processed_dict, save_path, metadata={"format": "pt"})
    else:
        torch.save(processed_dict, save_path)

存盘原语torch.distributed.checkpoint.save/load（torch 自带，checkpoint_manager.py:436/:390）—— 每 rank 一个 .distcp + .metadata
state_dict值是 DTensor 的 local shard（1/N），经 get_model_state_dict（:101）取分片；全量从不物化
跨硬件GPU 的 dcp ckpt NPU 不能直用——dcp 含设备/布局元信息且依赖 dist 环境；必须先 merge_dcp_to_hf 回 HF 全量 safetensors 中转

小结　心智模型：FSDP2 的 ckpt 体系 = dcp = 分片 ckpt（每 rank 一份 .distcp + 一份 .metadata，torch.distributed.checkpoint 这个torch 自带模块的原语 dcp.save/load 落地，CheckpointManager（checkpoint_manager.py:285）只做组织——包 Stateful 适配器 ModelState/OptimizerState（:69/:162）、处理 extra_state、分流 sync/async、加 EP 维度 restore/drop）+ HF safetensors = 全量中转格式（训练前入料经 WeightConvAdapter 改键名加载，训练后产物经 merge_dcp_to_hf.py 离线合并写出，跨硬件迁移也得绕 HF 中转）。dcp 拓扑无关（8 卡存 4 卡可 resume），HF 通用但只含参数不含训练状态；两套各管一段，HF 是它们之间的中立桥梁。根目录 convert_ckpt_v2.py（推荐）/convert_ckpt.py（v1 即将废弃）则是另一套 Megatron 格式与 HF 之间的转换入口，与 FSDP2 dcp 体系并行。
常见误区：① 以为 FSDP2 的 dcp ckpt 能被 transformers 直接 from_pretrained 加载——不能，必须先 merge_dcp_to_hf 合并回 HF safetensors。dcp 存的是每 rank 分片 + DTensor 布局元信息，transformers 不认这个格式；merge_dcp_to_hf.py（:265 no_dist=True + :276 full_tensor()）才把分片拼回全量写成标准 safetensors。② 以为 dcp 和 torch.save 一样是单文件全量——dcp 是每 rank 一个 .distcp + 一份 .metadata（checkpoint_manager.py:570 single_file_per_rank=True），存的是 1/N 分片不是全量。③ 以为 dcp 恢复必须同卡数——dcp 拓扑无关，dcp.load（:390）按当前 process_group 的 rank 数 reshape 分片，8 卡存的 4 卡可 resume。④ 以为 merge_dcp_to_hf 要把整个模型加载进内存——它只读 metadata 规划分片（:141 read_metadata() 不读张量），逐 shard load + 立即清显存（:286），内存峰值 = 单 shard 不是整个模型。⑤ 以为 merge_dcp_to_hf 不传 --model-configs 也能直接 from_pretrained——不能，产物只有权重没有 config.json/tokenizer.json（官方文档 :97 反复强调），必须显式传 --model-configs 拷贝。⑥ 以为 convert_ckpt_v2.py 和 merge_dcp_to_hf.py 是同一套——不是，V2 处理 Megatron 格式↔HF（:148 Hf2MgConvert / :150 Mg2HfConvert，含 TP/PP/EP 重切），merge_dcp_to_hf 处理 FSDP2 dcp→HF；FSDP2 用户走 merge_dcp_to_hf。⑦ 以为 checkpoint 这摊代码也像 EP08/EP10 那样主体在外部 fsdp_turbo——不是，dcp 原语是 torch 自带（:15-24 import），CheckpointManager 主体在仓库内，只有少量 EP 工具（restore_ep_dim/drop_ep_dim，utils.py）是仓库自研但也在 checkpoint/ 目录下。
