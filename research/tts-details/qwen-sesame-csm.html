Qwen2.5-Omni / Sesame CSM：端到端模型需要外部控制层

这类模型不是 multi-agent 系统，但它们把语音理解和语音生成推向端到端。产品上更合理的做法，是让 agent runtime 管理目标、工具、记忆、合规和评估。

对象：端到端语音/多模态模型 结构：内部模块化 + 外部 agent runtime 适用：实时语音助手、陪伴、客服
摘要 对比 外部控制层 局限 来源

30 秒摘要

Qwen2.5-Omni

公开介绍强调端到端多模态、实时语音输出，以及 Thinker-Talker 一类分工：一个模块偏理解和推理，一个模块偏流式语音生成。

Sesame CSM

Sesame 将 conversational speech generation 放在核心位置，使用对话上下文生成更自然的语音，并公开讨论了人类对话结构仍难完全建模的问题。

与传统 TTS 链路的区别

链路 优势 问题
ASR → LLM → TTS 模块清晰，容易替换和观测。 延迟高，语音情绪和对话节奏在模块间损失。
端到端 speech model 更低延迟，更容易保留语音情绪和对话上下文。 局部问题难诊断，工具调用、合规、记忆和回滚仍需外部系统。
端到端模型 + agent runtime 模型负责自然交互，runtime 负责目标、工具、策略和评估。 需要清晰边界：哪些交给模型即时处理，哪些必须由系统约束。

推荐的 agentic 外壳

1

Policy Agent

控制敏感场景、身份边界、拒答和安全策略。

2

Memory Agent

管理长期偏好、角色关系和会话摘要。

3

Tool Agent

处理 RAG、API、事务、日历、CRM 等外部动作。

4

Voice Critic

监测打断、情绪过度、幻觉和用户不适信号。

5

Fallback Router

在端到端模型失败时切回文本链路或人工接管。

局限

端到端不等于端到端可控

模型可以更自然，但产品仍要知道它为什么这样说、能否调用工具、是否越界、是否该重试。越端到端，越需要外部观测和策略层。
