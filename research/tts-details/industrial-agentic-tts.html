工业 TTS 正在从“音频 API”变成 agentic interface

OpenAI、Google、ElevenLabs、AWS、Cartesia 的共同趋势是：更自然的风格指令、更流式的输出、多说话人能力、情绪/标签控制和语音对话集成。

5 个能力信号

信号 代表来源 对 multi-agent TTS 的意义
自然语言风格指令 OpenAI TTS、Google Gemini TTS Prosody Director 可以输出 instructions，而不只输出 SSML。
多说话人和 dialogue Gemini TTS、Eleven v3 Voice Casting 可以把角色和声音资产显式绑定。
情绪/非语言标签 Eleven v3、Cartesia Sonic 导演 agent 可以用标签控制笑声、耳语、叹气、兴奋等表现。
低延迟流式 OpenAI Realtime、Cartesia、AWS Nova Sonic Synthesis Router 要按实时/离线制作分流，而不是单模型覆盖。
语音对话 + 工具 AWS Nova Sonic、OpenAI Realtime TTS 进入 voice agent runtime，和工具调用、RAG、电话集成耦合。

建议的模型路由策略

实时对话

优先低延迟、打断处理和工具调用。音质可以略低于离线制作，但稳定性和 time-to-first-audio 更关键。

高表现力内容

优先情绪、标签、多说话人和可编辑性。适合播客、有声书、广告、角色剧。

企业合规场景

优先权限、日志、授权声音、区域合规、可追溯和可回滚，性能声明必须在内部语料上复测。

工业栈里的 agent 分工

不要把供应商 API 当作完整产品

API 负责合成能力，产品还需要 intent planner、script agent、voice casting、prosody director、policy agent、critic、router、observability。agentic TTS 的价值就是把这些职责外显。
