Harness Engineering

Harness 工程

如果要把 DAW agent 做成能长期迭代的工程，harness 比模型 prompt 更关键。它负责把创作任务变成可重放、可比较、可恢复的实验。

最小可用 Harness

模组职责第一版交付
Fixture Manager维护可重放工程：空工程、pop vocal、多轨鼓、MIDI 编曲、混音 session。每个 fixture 有 project copy、expected state、render reference。
State Snapshot序列化 track、clip、MIDI notes、devices、parameters、routing、tempo、markers。JSON snapshot + hash + readable summary。
Action Ledger记录 agent plan、工具调用、DAW response、warnings、undo token。单次 run 产生一个 ledger.json。
Dry-run/Diff写入前预览 affected objects、风险和可回滚性。HTML diff report。
Undo / Rollback把多个低阶 API call 分组，并准备可恢复策略。REAPER: undo block + snapshot；Ableton: host undo/backup project；通用: fixture copy rollback。
Render & Audio Diff对变更前后 render 做 loudness、peak、spectral、silence、duration 检查。基础 LUFS/peak/duration + waveform hash。
Human Gate审批 destructive/expensive/opaque 操作。CLI/HTML approval prompt，含 diff 和 rollback。

Evaluation Protocol

层级指标外部参考
结构正确track/clip/device/marker 是否符合 typed plan；schema validation；snapshot diff。自建 fixture corpus、DAWProject/RPP/Ableton parser。
音频安全duration、peak、LUFS、silence、clipping、NaN/denormal、render reproducibility。audioldm_eval、Stable Audio Metrics、FAD 类方法。
Prompt adherence是否满足 bar count、style、instrumentation、mix intent。MusicBench/Mustango、CLAP-style audio-text similarity 可作辅助，不可替代人工听感。
音乐质量节奏、和声、编曲、混音自然度、过度处理。MIREX/MARBLE 等 MIR 评估传统 + 人类偏好评测。
可恢复性undo 成功率、rollback 成功率、失败后 project 可打开。每次 release 的 fixture regression。

评估任务集

任务成功条件自动检查人工检查
建立 8-bar drum groove指定 tempo、bar length、kick/snare/hihat pattern 存在。MIDI note grid、clip length、track count。节奏是否像 prompt 要求。
把 vocal 做得更暖新增或调整 EQ/saturation/compression，保留可撤销操作。device/parameter diff、peak 不爆、undo 成功。声音是否符合「warm」语义。
生成 song arrangementIntro/Verse/Chorus/Bridge marker 和 clips 分布合理。marker names、bar ranges、clip density。段落是否有音乐逻辑。
整理 sessiontrack 命名、分组、颜色、路由符合规则。naming lint、routing graph、no orphan tracks。是否提高可读性。
reference mix matching朝 reference 的 loudness/spectrum 靠近，但不破坏素材。LUFS、spectral centroid、dynamic range。是否过度处理。

安全分级

等级例子策略
Low读取 track list、播放状态、tempo、clip metadata。允许自动运行，记录即可。
Medium新增 MIDI clip、调整非破坏性 device parameter、建立 markers。dry-run + 可撤销 transaction。
High删除 track、覆盖 audio file、批量 normalize、下载外部素材、运行任意脚本。人类审批 + project backup + allowlist。
Blocked by defaultshell 任意命令、读取私人目录、上传 stems、安装插件。禁用，除非明确授权并隔离运行。

Security Harness

测试目标范例
MCP authorization验证 resource/audience、scope、per-client consent；拒绝 token passthrough。素材服务 token 不得被 DAW MCP server 当成自身 token 使用。
Local sandbox限制 project root、plugin path、render output、shell 命令。agent 只能读写 fixture 目录，不能扫描使用者整个音频库。
SSRF / URL policy外部素材、参考曲、plugin installer URL 需 allowlist/denylist。拒绝 localhost metadata、file://、私有网段下载。
组合风险检查多个低风险工具连用后是否变成数据外泄或 RCE。filesystem + Git + plugin install + DAW script 需要更高 gate。

Versioning Fixture Policy

每个 fixture 应包含 project copy、media manifest、plugin manifest、preset manifest、expected snapshot、render reference 与最小可重放命令。大型 stems 用 Git LFS 或外部 artifact store，project file diff 只作静态信号，最终仍需宿主加载和 render 验证。

Plugin / Model Regression

插件与模型工具应测试 scan/load/unload、参数 automation、offline render determinism、CPU/GPU fallback、missing plugin fallback、模型权重 hash、license/provenance 和 crash recovery。这一层尤其适合把 VST3/AUv3/AAX/CLAP/JUCE 与 Neutone/DDSP/RAVE 类模型分开建 fixture。

DAW 特定落地

REAPER

较适合先做 harness。ReaScript 可用 Lua/Python/EEL 编写单次 mutation 脚本，官方文档说脚本可呼叫 REAPER action 和多数 REAPER API functions；是否具备 atomic/rollback 语义，取决于 harness 的错误处理、状态快照和恢复策略。

Ableton

AbletonOSC/Remote Script 路线可快速 demo；Extensions SDK 路线更值得长期押注，但截至 2026-06 仍是 Live 12.4.5 Suite beta。harness 应保留多 bridge adapter，避免过早绑死。
