研究问题

当 LLM 被接进 DAW，它究竟应该做什么？本调研把任务拆成五层：读取 session 状态、生成可审批的操作计划、运行 DAW mutation、回传可验证结果、把结果纳入创作者下一轮意图。

技术地图

Host API 层

REAPER ReaScript/API、Ableton Live Object Model/Remote Script、Ableton Extensions SDK、FL Studio Python MIDI scripting、Bitwig controller extension API、Logic/Cubase 的 AppleScript/MIDI/control surface 通道。

Transport 层

MCP stdio/HTTP、OSC、MIDI、WebSocket、local socket、filesystem/project-file diff。传输层不能替代领域安全层。

Agent 能力层

session 查询、arrangement 生成、clip/MIDI 编辑、mixing macro、device parameter editing、render/export、素材检索与 reference matching。

Plugin / Model 层

VST3、AUv3、AAX、CLAP、JUCE 与神经音频插件会把 agent 风险带到插件扫描、模型权重、即时延迟、离线 render 一致性、版权与供应链。

Harness 层

state snapshot、tool schema、dry-run、mutation log、undo transaction、audio render diff、human approval、evaluation corpus。
