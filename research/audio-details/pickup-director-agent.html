背景介绍

从固定拾音规则到“该听谁”的导演决策

传统拾音系统通常围绕波束形成、降噪和回声消除展开，目标是让声音更干净。但真实场景的问题往往更语义化：会议中此刻该听主讲人还是提问者，直播中要保留多少现场欢呼，课堂里要不要捕捉学生插话，采访时哪些环境声是氛围而不是噪声。智能拾音导演 agent 的价值，是把声学信号处理与任务目标结合起来。

近年的 audio-visual target speaker extraction 研究已经开始把视觉、语言和目标约束引入分离过程。语言约束可以告诉系统“提取正在解释产品的人”，视觉约束可以帮助判断嘴型和方位，麦克风阵列提供方向与距离。agent 层负责把这些线索组织成可解释的拾音策略。

使用场景

拾音成为内容生产和协作的前置智能

  • 会议室与远程会议：根据议程和发言权重动态聚焦当前有效说话人，同时保留打断与附和。
  • 直播与采访：在主讲、人群声、环境声之间动态平衡，避免把现场感完全消掉。
  • 课堂录制：老师、学生提问和板书声都可能重要，agent 根据教学阶段调节拾音目标。
  • 法庭/听证记录：优先保障发言归属、低误删和可追溯原始音频，而不是追求过度降噪。
关键技术创新

把拾音变成可解释的多目标优化

  • Semantic Target Selector：将“听主持人”“保留观众反应”“忽略侧后方闲聊”等自然语言目标转成分离约束。
  • Audio-Visual Fusion：结合嘴型、人体朝向、声源方向和声纹线索，提高嘈杂场景下目标锁定稳定性。
  • Scene-aware Noise Policy：区分应保留的环境声和应抑制的干扰声，避免把氛围当噪声清掉。
  • Pickup Decision Log：记录每次聚焦、衰减、切换的理由，便于直播、会议和法务场景复核。
  • Fallback Raw Track：始终保留原始多轨或阵列信号，防止 agent 错删不可恢复信息。

MVP/实验

先做“会议拾音导演”

MVP 可用 4 麦阵列和摄像头，处理 20 场模拟会议：主讲、插话、多人重叠、远端噪声和投影仪风扇。指标包括目标说话人保真、非目标抑制、插话保留率、speaker attribution accuracy、听感偏好和原始轨可恢复率。失败信号是系统为了干净而删掉关键插话，或在多人重叠时频繁切错目标。

产品化时先提供“导演模式”选择：会议记录、直播现场、采访、课堂。每个模式有不同保留策略，而不是同一套降噪参数。用户还应能回看拾音决策日志，知道为什么某段环境声被保留或抑制。

风险

不能让 agent 成为不可见剪辑师

拾音 agent 一旦错删，后期很难恢复。严肃场景必须保存原始轨，并明确标记哪些音频经过增强或分离。隐私也是核心风险：摄像头、声纹和会议内容都可能敏感，默认应本地处理或最小化上传。
