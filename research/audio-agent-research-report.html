关键发现

Salesforce Research（2026.03）的关键结论：尽管原生 S2S 模型架构更优雅，但当前所有生产级语音 Agent 都使用级联管道。原因是原生 S2S 模型（1）不支持函数调用，（2）延迟反而更差（Qwen2.5-Omni 的 13s TTFA 是级联方案的 17 倍）。级联方案通过流式 + 句子缓冲实现伪实时。

IBM 递归式长音频生成
Self-Correcting

关键缺口

现有基准全部聚焦于音频感知/理解能力——即模型"能不能听懂"。但没有一个基准评估 Agent 控制音频生产工具的能力——即"能不能干活"。这类似于 NLP 领域从 MMLU（知识评估）到 SWE-Bench（工程能力评估）的演变——音频领域亟需自己的"SWE-Bench"。

ReAct 循环 (Reason+Act+Observe)
来自 Coding Agent

源头：Devin、SWE-Agent 将任务建模为 DAG，逐步执行并动态重规划。Devin 2.2 增加自验证和自修复能力。

音频迁移：混音 Agent 使用 Plan-Mix-Listen-Remix 循环——推理"人声被吉他遮蔽在 2-4kHz" → 应用侧链动态 EQ → 通过频谱分析验证 → 若仍遮蔽则重规划（尝试音量自动化或声像调整）。这是第一个真正的迭代式 AI 混音助手，而非当前的一次性预设推荐。

高新颖度 混音迭代
Agent-DAW Interface (ADI)
来自 SWE-Agent ACI

源头：SWE-Agent 定义了 Agent-Computer Interface（ACI），标准化 LLM 与代码库的交互命令，无需领域特定微调。

音频迁移：定义 Agent-DAW Interface——search_tracks(filter)、apply_plugin(track, plugin, params)、analyze_spectrum(track, range) 等标准命令集。将 AI 音频 Agent 从特定 DAW API 解耦，写一次 Agent，在任何 DAW 运行。已有早期种子：MCP server for Ableton Live。

方向 1：音频 Agent 操作系统

类比操作系统管理进程和内存，构建一个"音频 Agent 操作系统"——管理多个音频 Agent 的生命周期、资源分配、进程间通信。用户只需描述高层意图，系统自动编排合适的 Agent 组合。

关键技术：Agent-DAW Interface (ADI)、Letta 记忆管理、LangGraph 状态编排

突破性 系统级创新
方向 2：涌现式音乐创作系统

每个"乐器 Agent"遵循简单音乐规则（调性、节奏、动态），通过交互产生复杂音乐编排。不预设曲式，音乐形式从 Agent 间交互中涌现。像乐队即兴，但每个"乐手"都是 AI。

关键技术：集群智能、多 Agent 群聊、简单规则涌现

突破性 创作范式革新
方向 3：记忆驱动的音频个性化引擎

四层记忆系统（短期/情景/语义/程序）驱动的音频生产助手。随着时间推移学习用户的音色偏好、工作流习惯、决策模式。从"工具"进化为"伙伴"。

关键技术：MemGPT 三层记忆 + Stanford Generative Agents 反思机制

快速反应层（亚毫秒级音频处理）+ 慢速规划层（前瞻性决策）的双层架构。

高价值 实时交互 全双工
