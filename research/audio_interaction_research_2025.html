🧠 通用 Audio Encoder & 自监督预训练

从 Wav2Vec 2.0 到 USAD，自监督语音预训练经历了从单模态到跨模态、从通用到领域专精的演进。Neural codec 和可学习前端也在重新定义音频表征的边界。

🔬 Neural Audio Codec 进化

Neural codec 从音频压缩工具演变为语音 AI 生态的基础设施，支撑 TTS、音乐生成、语音增强等多个下游任务。

EnCodec Meta

AudioCraft 生态核心 codec，支撑 MusicGen 和 AudioGen 生成。基于残差向量量化（RVQ），多种比特率可选。

SoundStream Google

首个在智能手机 CPU 上实时运行的 neural codec，已用于 NotebookLM 播客生成。开创了 neural codec + 生成模型的技术路线。

DAC Descript

44.1kHz 以 ~90x 压缩率编码为 8kbps tokens，改进的 RVQ + snake activation，显著提升音频重建质量。

DACe 2025

DAC 增强高保真版本，进一步提升 codec 的音频还原质量，适用于对音质要求更高的场景。

SpectroStream 2025

新一代 neural codec，支持 full-band 48kHz，在保持低延迟的同时实现高保真音频编解码。

📈 自监督语音模型进化路线

从 contrastive learning 到跨模态统一框架，自监督语音预训练的范式不断演进。

Wav2Vec 2.0
基础模型 — Masked prediction + contrastive learning + quantization，开创了自监督语音预训练的主流范式。
HuBERT
简化训练 — 用 clustered hidden units 的 masked prediction 替代 contrastive loss，训练更稳定，性能更强。
WavLM
鲁棒性提升 — 加入 denoising + speaker discrimination 多任务，在噪声环境下的表现显著提升。
Data2Vec (Meta)
跨模态统一 — 跨模态（语音/视觉/文本）统一自监督框架，同一个架构处理不同模态。
BEATs (Microsoft, ICML 2023)
BERT 风格音频预训练 — 将 NLP 中成功的 BERT 模式引入音频领域，迭代式 tokenizer + masked prediction。
OpenBEATs (2025)
完全开源扩展 — 完全开源多领域 BEATs 扩展，覆盖 25 个 benchmark，支持语音、音乐、环境音。
USAD (MIT CSAIL, 2025)
统一 encoder — 多领域 SSL 蒸馏为统一 encoder，一个模型覆盖语音、音乐、环境声等多领域。
UniWav (ICLR 2025)
统一预训练框架 — 统一多种语音任务（ASR、SV、SD 等）的预训练框架，减少任务特定微调需求。

⚡ 边缘端音频特征提取

在 MCU 等资源受限设备上，特征提取的计算效率直接决定系统能否实时运行。

MFCC / Mel-filterbank

仍是 edge 端主流选择，计算量最小，硬件加速友好，几乎所有 DSP 库都原生支持。

SincNet

参数化 band-pass filters，可学习，适合 MCU 部署。直接从波形学习 filterbank 参数。

LEAF (Google)

全可学习前端，Gabor filters + Gaussian pooling，端到端优化特征提取。

Faster LEAF

仅 3% 计算量即达 LEAF 精度，大幅降低可学习前端的部署门槛。

重要发现 ：可学习前端在多数场景下不能一致性地超越固定 mel-filterbank baseline。在资源受限的 edge 端，mel-filterbank 仍是性价比最高的选择。

👥 Speaker Diarization

说话人日志（"who spoke when"）在会议、播客、客服等场景中是核心上游组件。

Pyannote 3.1

最主流开源 diarization 工具，DER 约 10%。基于 segmentation + clustering 流水线，社区生态完善。

SpeakerLM

使用 LM 进行端到端 diarization 的创新方法，将说话人识别和日志统一为序列建模任务。

Meta SVoice (2020)

单麦分离 5 个同时说话人，突破了传统方法的说话人数量上限，在鸡尾酒会场景下表现优异。

🔬 SALMONN-omni NeurIPS 2025

不将 audio codec 注入 token vocabulary，而是以独立 speech LLM 实现全双工对话。这是2025年音频语言模型的重要里程碑 —— 首个无需文本中间表示即可实现听+说并行处理的 speech LLM。

量化方法对比 范式

不同的量化方法决定了 Codec 的压缩效率、训练稳定性和对 LLM 建模的友好程度。

RVQ — Residual Vector Quantization

迭代量化残差：第一个 codebook 捕获粗粒度特征，后续 codebook 逐层精细化残差。比特率可通过 codebook 数量灵活控制。

  • 优势：精细比特率控制，压缩质量可控
  • 劣势：Codebook collapse 问题（部分码字不被使用）
  • 代表：EnCodec, DAC, Mimi, SpeechTokenizer, DualCodec

FSQ — Finite Scalar Quantization

每个维度独立量化为有限集合中的值，无需 codebook lookup。bit 级鲁棒性优越，训练更稳定。

  • 优势：无 codebook collapse，bit 级鲁棒性
  • 劣势：表达上限受维度数约束
  • 代表：X-Codec-2, NeuCodec

Single VQ — 单 Codebook 量化

使用单个大 codebook（如 4096 entries），极大简化 LLM 建模——LLM 只需在一个离散空间上建模。

  • 优势：LLM 架构极简，无需 inter-codebook 建模
  • 劣势：信息瓶颈更紧，重建质量受限
  • 代表：WavTokenizer

Binary VQ — 二进制量化

极致压缩路线：使用单个 binary codebook，每个 token 仅 1 bit 信息量。适合超低比特率场景。

  • 优势：理论压缩极限
  • 劣势：重建保真度有限
  • 代表：FocalCodec

比特率压缩趋势 里程碑

从 2021 年 SoundStream 的 3 kbps 到 2025 年 FocalCodec 的 0.16 kbps，比特率压缩近 20 倍，核心驱动力：更高效的量化方法 + 更低帧率 + 语义先验注入。

语义增强三阶段演进 关键洞察

Neural Codec 的核心演进不在压缩率，而在"语义信息"的引入方式——从纯声学到蒸馏注入再到架构级集成。

第一代 (2021–2022)
纯声学编解码

纯重建目标，无语义信息。Codec 只负责将波形压缩为 token 并还原，token 空间与语义空间无对齐。

  • SoundStream — RVQ 纯声学
  • EnCodec — RVQ + 多尺度 discriminator
无语义对齐 纯重建 MSE/Mel

第二代 (2023–2024)
Distillation 注入语义

将预训练 SSL 模型（HuBERT, WavLM）的语义信息蒸馏到 RVQ 的第一层 codebook，使第一层 token 具备语义表征能力。

  • SpeechTokenizer — HuBERT → RVQ-1 蒸馏
  • Mimi — WavLM → RVQ-1 蒸馏
  • X-Codec — HuBERT 拼接到编码空间
蒸馏损失 RVQ-1 语义化 语义-声学解耦

第三代 (2025)
架构级语义集成

语义不再是"外挂"的蒸馏目标，而是内建到 Codec 架构本身——独立语义流、attention 机制或新量化范式自带语义。

  • DualCodec — 独立 SSL 编码流 + 声学流双流架构
  • WavTokenizer — Attention 内建语义捕获
  • X-Codec-2 — FSQ 量化本身具备语义表达力
架构内建 无外部蒸馏 端到端语义

连续 vs 离散 Token 对比

连续 Token（如 Whisper encoder 输出）

  • 保留更丰富的声学信息，信息无损
  • 需额外 adapter 层与 LLM 对齐（如 Q-Former、线性投影）
  • 计算量较大，无法直接复用 LLM embedding 层

离散 Token（如 EnCodec / Mimi 输出）

  • 可直接接入 LLM embedding 层，复用标准 transformer 架构
  • 量化损失不可避免，高频细节可能丢失
  • 天然支持与文本 token 交错，便于多模态融合
