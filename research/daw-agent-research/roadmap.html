Roadmap

落地路线图

一个务实的 DAW agent 原型应先证明「可控修改」而不是「看起来聪明」。下面是 30/60/90 天工程路线。

首页生态项目DAWZY 深挖代码解剖架构Harness来源

30 天：可观测 MVP

交付内容验收
选宿主建议先选 REAPER 或 Ableton。REAPER 便于 harness；Ableton 便于社区/产品展示。能读取 project/session state。
10 个高语义工具create track、create MIDI clip、set tempo、insert marker、set volume、add simple device、render、snapshot、dry-run、undo。每个工具有 schema、风险等级、单元测试。
授权与沙箱MCP scope、resource/audience、tool allowlist、project root sandbox、外部 URL policy。高风险工具预设被 gate；token passthrough/越权路径测试失败即阻断。
Fixture harness3 个 project fixture + snapshot/diff/ledger。CI 或本地命令可重放 5 个任务。

60 天：创作工作流

交付内容验收
Planner自然语言 → typed edit plan；禁止模型直接运行低阶 API。计划可 dry-run 和人工 approve。
音乐任务集drum groove、bassline、arrangement、mix cleanup、vocal warm-up。自动 metrics + 人工评分表。
权限模型tool allowlist、project sandbox、外部 IO policy。高风险操作必须被 gate 挡住。

90 天：多宿主/产品化

交付内容验收
Adapter abstraction把 host-specific code 藏在 adapter，工具语义保持一致。同一任务可在两个宿主或一个宿主 + openDAW fixture 上跑。
Evaluation dashboard显示任务成功率、undo/rollback 成功率、音频安全、prompt adherence、人工评分。每次 release 有结构、音频、偏好三层 regression report。
Creative UXpreview、accept/reject、explain changes、before/after render。创作者能安全迭代，不需要信任黑盒 agent。

优先级建议

  1. 先做 REAPER harness。DAWZY 和 REAPER MCP 生态提供了可逆脚本路线的初步可行性证据，ReaScript 的工程可控性也相对好。
  2. 同步跟 Ableton Extensions SDK。它可能成为 Ableton agent 化的长期官方入口，但 beta 期不要把核心架构绑死。
  3. 避免工具爆炸。与其暴露 400 个低阶工具，不如 20 个高语义工具 + 可靠 planner + 可重放 eval。
  4. 把「听起来好」拆成两层。先用客观 audio metrics 防止坏结果，再用人工或偏好模型评估音乐审美。
Roadmap built from ecosystem findings and harness risk analysis; priorities are recommendations, not proof of production readiness.
