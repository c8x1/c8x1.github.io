REPORT 04

🤖 AI Agent 与 DAW 控制

DAWZY、ReaperAI、AI DAW 等项目的 Agent 架构深度分析

⭐ DAWZY — 学术级 DAW Agent 先驱

📦 San Diego State University / JSBAI Center 👤 Aaron Elkins et al. 📅 2025 年 12 月 📄 学术论文 (NeurIPS 2025 Workshop)
LLM Code Generation REAPER / ReaScript Electron Gateway MCP Tools Voice-first

DAWZY 是目前最重要的 DAW Agent 学术研究项目 (NeurIPS 2025 Workshop)。它提出了一种新颖的方法： 使用 LLM 代码生成（而非直接的 API 调用）来控制 REAPER DAW， 从而显著减少了 LLM 幻觉（hallucination）问题。

核心架构

Processing Layer

用户意图（文本/语音/哼唱）通过 Electron Gateway 流向 LLM 和 MCP Tools， 然后以可逆的 ReaScript 脚本在 REAPER 中执行。

  • 输入层: 支持文本、语音、哼唱等多种输入方式
  • Electron Gateway: 作为用户界面和协议层
  • LLM 引擎: 理解用户意图并生成 ReaScript 代码
  • MCP Tools: 提供 DAW 状态查询和上下文信息
  • 执行层: 将生成的代码作为可逆操作在 REAPER 中执行

关键创新

  • LLM 代码生成减少幻觉: 相比直接调用 API，让 LLM 生成可执行的 ReaScript 代码， 可以更容易验证操作的正确性，显著降低了 "幻觉命令" 和 "错误索引" 问题。
  • 可逆操作: 所有 Agent 执行的操作都是可逆的（可通过 Undo 回退）， 给用户安全感，鼓励实验性使用。
  • 上下文感知: Agent 可以查询当前 DAW 会话状态， 使操作更加精确和相关。
  • Voice-first 设计: 优先考虑语音交互，适合音乐制作场景。
  • 权限化函数: DAW 能力被暴露为显式的、有权限控制的功能接口。

与其他项目的对比

"DAWZY primarily interacts with REAPER through LLM code generation — a more powerful approach than similar tools like Ableton-MCP."

— DAWZY 论文, arXiv:2512.03289

🔧 ReaperAI

📅 2026 年 1 月 🧪 社区项目

ReaperAI 是一个专注于 REAPER DAW 的 LLM 自动化助手项目。 它的目标是减少 REAPER 中复杂/重复操作的摩擦，包括：

  • 轨道组织和命名管理
  • 信号路由配置
  • Action/脚本编排
  • 项目配置自动化

项目强调通过 RAG（Retrieval-Augmented Generation）加载 REAPER 文档， 使 Agent 能够准确理解 REAPER 的 API 和功能，适合初学者使用。

🏗️ Building a Simple AI DAW (Johannes Naylor)

👤 Johannes Naylor 📅 Part 1: 2025.01 | Part 2: 2025.08
Python Rust LangChain Claude FFmpeg MCP

这是一个技术教程系列，展示了从零构建 AI 音频 Agent 的完整过程。

Part 1: Python + LangChain 方案 (2025.01)

使用 Python + LangChain 构建一个 AI 音频操作 Agent。Agent 接收用户的自然语言请求， 让 Claude 生成 FFmpeg 命令来处理音频文件。这是一个简单但有效的起点。

Part 2: Rust 流式引擎 + MCP (2025.08)

第二部分是技术深度的飞跃。核心升级包括：

  • Rust 流式引擎: 替换 Python/FFmpeg 方案，用 Rust 实现高性能、 低延迟的实时音频流处理
  • MCP 协议集成: 将 Agent 能力暴露为标准 MCP 接口， 可与 Claude Desktop、Cursor 等 MCP 客户端集成
  • 更健壮的 Agent 架构: 从简单的"生成命令"模式升级为 具有状态管理和错误恢复的完整 Agent

🖱️ "Cursor for DAW" 概念实验

👤 Xiaohan Tian 📅 2025 年 8 月

一篇探索性文章，讨论如果将 Cursor（AI 代码编辑器）的理念应用到 DAW 中会是什么样子。 作者构建了一个轻量级浏览器 DAW 实验原型，内嵌 AI 音乐家助手。

核心理念

  • AI 助手理解当前项目的音乐上下文（和弦进行、节奏、音色）
  • 可以像 Cursor 的 "Tab 补全" 一样，建议下一个音符、和弦或编曲元素
  • 支持自然语言指令来修改音乐内容
  • AI 和人类在同一时间线上协作

🎚️ AI Mixing Assistant Prototype (REAPER)

📅 2026 年 3 月 🧪 工作原型

在 Steinberg 论坛上（讨论 Cubase 的 AI 集成缺口时），有开发者提到了一个 针对 REAPER 的 AI 混音助手工作原型，具备以下能力：

  • 音频分析: 自动分析轨道音频的频率特征、动态范围等
  • 插件视觉: 能够 "看到" 当前加载的插件及其参数设置
  • 迭代式混音问题解决: Agent 能够识别混音问题（如频率冲突、过度压缩）， 提出修复建议并自动执行调整
  • Forum: Steinberg Forum Discussion

Agent 架构模式对比

项目 Agent 架构 LLM 用法 执行方式 可逆性
DAWZY Processing Layer + MCP 代码生成 (ReaScript) Lua/Python 脚本 ✅ Undo
ReaperAI RAG + ReaScript 文档检索 + 代码生成 ReaScript ✅
AI DAW (Naylor) Rust Engine + MCP 工具调用 (FFmpeg) Rust 流式引擎 部分
AbletonMCP MCP Server 直接 API 调用 Ableton API 部分
Mixing Assistant 分析-建议-执行 音频分析 + 决策 ReaScript ✅

🔑 Agent 架构关键洞察

  • LLM 代码生成是最可靠的 DAW 控制方式：DAWZY 的实验证明，让 LLM 生成脚本代码比直接调用 API 更不容易产生幻觉。
  • RAG + 文档检索提升了 Agent 的领域知识：ReaperAI 通过加载 REAPER 文档，使 Agent 能准确理解 DAW 的复杂 API。
  • Rust 引擎代表了性能升级方向：Johannes Naylor 的 Part 2 展示了从 Python/FFmpeg 到 Rust 流式引擎的演进路径。
  • 可逆操作是用户体验的关键：所有 Agent 操作都应该可以通过 Undo 回退，这大大降低了用户对 AI 错误的焦虑。
