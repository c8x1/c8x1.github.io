场景描述 & 痛点

场景：直播场景，主播佩戴领夹麦/胸麦 (lavalier/bodypack mic)
目标：检测主播是否在说话（flag → 后续算法处理依据）
干扰：附近路人经过时的说话声被麦克风收入，不能误检
当前痛点：使用 Silero VAD，主播语音与路人语音的 probability 差异太小，无法可靠阈值化
约束：模型大小需与 Silero VAD (~2MB) 同级别，轻量可实时运行

1 Personalized VAD / Target-Speaker VAD

Google Personal VAD (Odyssey 2020)

最直接相关
论文"Personal VAD: Speaker-Conditioned Voice Activity Detection"
作者Shaojin Ding, Enno Hermann 等 (Google)
核心思路在标准 VAD 基础上，使用目标说话人的 d-vector 对模型进行 conditioning，输出变为 3 分类：silence / non-target speech / target speech
模型架构基于 RNN/GRU 的 frame-level 模型，输入 log-mel spectrogram + d-vector conditioning
模型大小~150K–500K 参数（VAD 部分）；d-vector 提取为独立组件
开源非官方实现：pirxus/personalVAD
可行性：高 — 架构简洁，模型轻量，可直接应用于直播场景。只需预录制几秒主播语音生成 d-vector。

TS-VAD (Interspeech 2020)

奠基论文
论文"Target-Speaker Voice Activity Detection: A Novel Approach for Multi-speaker Diarization"
核心思路将每个说话人的 embedding 作为输入，用神经网络预测每帧每个说话人的 VAD 状态
局限原始设计面向离线 diarization，需先聚类获取 speaker profiles，不太适合实时单目标场景

Noise-Robust TS-VAD via SSL + FiLM (AAU, 2025)

关键技术
论文"Noise-Robust Target-Speaker Voice Activity Detection Through Self-Supervised Pretraining"
arXiv2501.03184
核心贡献提出 DN-APC 自监督预训练框架；系统对比确认 FiLM conditioning 效果最优
性能提升seen 和 unseen noise 条件下均提升约 2%
FiLM (Feature-wise Linear Modulation) 是目前 pVAD 领域效果最好的 conditioning 方法：
speaker_embedding = encoder(enrollment_audio) # 256-dim gamma, beta = FiLM_layer(speaker_embedding) # 对每个音频帧的中间特征进行调制 frame_feature = backbone(audio_frame) modulated = gamma * frame_feature + beta output = classifier(modulated) # 3-class: silence / non-target / target

pVAD Comparative Analysis (Apple Research, 2024)

必读综述
arXiv2406.09443
核心价值系统比较多种 pVAD 变体，提出 frame-level/utterance-level 误差率、检测延迟、准确率等多维评估框架

Robust pVAD for Domain Mismatch (Interspeech 2025)

最新进展
核心方法结合 embedding-free processing + dynamic enrollment 技术
解决的问题训练/部署 domain mismatch 和 false acceptance — 正是直播场景的核心痛点

2 Lightweight Speaker Verification as VAD Prefilter

Resemblyzer

原型验证首选
仓库resemble-ai/Resemblyzer
核心基于 d-vector 的 speaker embedding 提取，输出 256 维 embedding
模型大小~15 MB（偏大，但适合快速验证）
开源MIT License
优点接口极简，几行代码即可提取 embedding

CAM++ (Alibaba DAMO)

速度精度兼顾
仓库modelscope/3D-Speaker
BackboneD-TDNN (Densely-connected Time Delay Neural Network)
参数量~7.2M 参数，~27 MB
性能比 ResNet34 和 ECAPA-TDNN 更快更准
量化后INT8 ~7MB，再加剪枝 ~3-4MB，接近目标

Asymmetric Enroll-Verify Architecture

推荐思路
核心思路：enrollment 时用服务端高精度大模型提取 voiceprint embedding，verification 时用极小模型做 cosine similarity。

非常适合直播场景：主播注册用服务端大模型，本地用极小模型实时验证。两阶段不对称，既保证 enrollment 质量又保证推理速度。

WeSpeaker (wenet-e2e)

部署链路最成熟
仓库wenet-e2e/wespeaker
支持模型X-vector, ResNet34, ECAPA-TDNN, CAM++
ONNX 支持完整，通过 sherpa-onnx 全平台部署
开源Apache 2.0

3 Direction-of-Arrival (DOA) Based Approaches

领夹麦的天然优势：主播语音来自近场 (~10-20cm)，路人语音来自远场 (~1-5m)。这种近/远场差异可作为 零成本辅助特征，但不能作为唯一判据。
特征近场 (主播 ~10-20cm)远场 (路人 ~1-5m)
直达/混响比 (DRR)高低
Spectral Tilt低频增强 (proximity effect)正常衰减
信噪比 (SNR)高低
Signal Energy强弱

优化策略

  1. VAD first, then SV — 先过滤非语音段，再做 speaker verification
  2. Sliding window — 固定窗口 (500ms–1s) 滑动提取 embedding
  3. Early rejection — VAD 概率过低直接跳过 SV
  4. Embedding caching — 缓存最近的 embedding 避免重复计算
  5. Temporal smoothing — 对连续帧的 cosine similarity 做平滑减少误判

7 TinyML / Edge-Optimized Speaker Models

TinySV (2024)

最符合约束
arXiv2406.01655
仓库AI-Tech-Research-Lab/TinySV
架构两层级联：Keyword Spotting (KWS) + Adaptive Speaker Verification
目标设备Arduino Nano 33 BLE 等极度资源受限 MCU
模型大小< 1 MB
特点支持 on-device learning，可在设备上更新 speaker model
可行性：极高 — 这是目前最接近 "Silero VAD 大小 + target speaker" 的方案。MCU 上都能跑，PC/手机端更无压力。

TinyCNN for ESP32 (2025)

方法量化 CNN 部署在 ESP32 MCU 上
精度98.58%（量化后）
性能比原始模型快 53%，小 73%

量化策略参考

技术压缩比精度损失
INT8 量化4x (float32 → int8)<1% EER 增加
INT4 量化8x1–3% EER 增加
知识蒸馏2–10x取决于教师模型
结构化剪枝2–5x可忽略

例：CAM++ (27MB) → INT8 ~7MB → 剪枝 ~3-4MB → 接近 2MB 目标
