### 原理图阅读基础

**原理图（Schematic）** 是电路设计的"蓝图"，用标准符号表示电子元件和连接关系。

    │  │0-39│ │   │ │  │ │   │ │   │  │

### PCB（印刷电路板）基础

```
PCB 层数：

2层板（本项目使用）：
  ┌─────────────┐
  │ 顶层 (Top)   │ ← 信号走线 + 元件焊盘
  ├─────────────┤
  │ 底层 (Bottom)│ ← 信号走线 + 地线铺铜
  └─────────────┘
  成本低、设计简单，适合简单电路

4层板（推荐用于生产）：
  ┌─────────────┐
  │ 顶层         │ ← 信号走线
  ├─────────────┤
  │ GND 层      │ ← 完整地平面（降低干扰）
  ├─────────────┤
  │ Power 层    │ ← 电源平面（稳定供电）
  ├─────────────┤
  │ 底层         │ ← 信号走线
  └─────────────┘
  更好的信号完整性、EMI性能

穿戴设备 PCB 特殊考虑：
  - 小尺寸：通常 < 50×20mm
  - 异形：跟随眼镜框形状
  - FPC（柔性 PCB）：可弯折，适合眼镜腿
  - 天线区域：保持净空，不能有铜皮
```

## 7.2 系统集成要点

### 功耗预算详解

```
功耗预算表（基于 300mAh 电池）

组件             │ 工作电流  │ 占空比  │ 平均电流  │ 备注
────────────────┼──────────┼────────┼──────────┼──────────────
ESP32 (240MHz)  │ 150 mA   │ 10%    │ 15 mA    │ 音频采集+处理
ESP32 (Modem)   │ 30 mA    │ 20%    │ 6 mA     │ WiFi空闲
ESP32 (Deep)    │ 0.05 mA  │ 70%    │ 0.035 mA │ 深度休眠
PDM 麦克风      │ 1 mA     │ 10%    │ 0.1 mA   │ 按需采集
复合视频输出     │ 50 mA    │ 20%    │ 10 mA    │ 按需显示
WiFi 发送       │ 170 mA   │ 5%     │ 8.5 mA   │ 音频上传
WiFi 接收       │ 120 mA   │ 5%     │ 6 mA     │ 接收结果
LED 指示灯      │ 5 mA     │ 5%     │ 0.25 mA  │ 状态指示
LDO 自耗        │ 5 mA     │ 100%   │ 5 mA     │ 不可避免
────────────────┼──────────┼────────┼──────────┼──────────────
合计             │          │        │ ~51 mA   │

续航时间 = 300mAh / 51mA ≈ 5.9 小时

优化后（添加 VAD + 智能休眠）：
  待机时：ESP32 Deep Sleep + BLE 广播 ≈ 0.5 mA
  活跃时：正常工作 ≈ 80 mA
  假设 80% 待机、20% 活跃：
  平均电流 = 0.8 × 0.5 + 0.2 × 80 = 16.4 mA
  续航 = 300 / 16.4 ≈ 18.3 小时

→ 翻了3倍！这就是 VAD + 智能电源管理的价值
```

### 热管理

```
ESP32 240MHz 满载功耗 ≈ 1W (3.3V × 300mA)

散热问题：
  - 1W 在 7×7mm 的芯片上 → 热密度很高
  - 穿戴设备不能有风扇 → 只能靠 PCB 铜皮散热
  - 温度 > 85°C 时 ESP32 会降频或复位

散热设计：
  1. 在 PCB 上芯片下方加散热铜皮
  2. 热过孔（Thermal Via）连接到地层
  3. 动态降频：空闲时降到 160MHz 或 80MHz
  4. 任务调度：避免两个核同时满载

  ESP32 动态频率调节：
  esp_pm_configure(&(esp_pm_config_t){
      .max_freq_mhz = 240,
      .min_freq_mhz = 40,   // 空闲时降到 40MHz
      .light_sleep_enable = true  // 启用轻度休眠
  });
```

### 可靠性设计

```
嵌入式系统的可靠性策略：

1. 看门狗定时器 (WDT)
   - 系统必须在超时前"喂狗"
   - 否则自动复位（重启）
   - 本项目：Bootloader WDT = 9秒
   - 建议添加 Task WDT：检测任务是否长时间阻塞

2. 掉电检测 (Brown-out Detection)
   - 电压低于阈值时自动复位
   - 防止电压不稳导致数据损坏
   - ESP32 默认 BOD 阈值 = 2.7V

3. OTA 回滚
   - 新固件更新后如果启动失败，自动回滚到旧版本
   - 防止"变砖"
   - ESP-IDF 原生支持

4. 错误处理
   - esp_error_check() 宏：失败时打印错误并重启
   - 自定义 panic handler：崩溃时保存日志
   - Core Dump：崩溃时保存寄存器和栈信息到 Flash
```

## 7.3 嵌入式AI前沿（面试加分项）

### TinyML 生态全景

```
TinyML 工作流（从训练到部署）：

┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  训练 (PC)    │────→│ 量化/优化     │────→│  转换        │────→│  部署 (MCU)   │
│              │     │              │     │              │     │              │
│ TensorFlow   │     │ INT8 量化    │     │ TFLite       │     │ TFLM Runtime │
│ PyTorch      │     │ 剪枝         │     │ .tflite      │     │ ESP-DL       │
│ 训练模型      │     │ 蒸馏         │     │ .espdl       │     │ 运行推理      │
│              │     │ 压缩         │     │ C 数组       │     │              │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘

模型精度（FP32）→ 量化后（INT8）：精度损失通常 < 1%
模型大小：FP32 → INT8：缩小 4 倍（32bit → 8bit）
推理速度：INT8 → 比 FP32 快 2-4 倍
```

### ESP32 上的 AI 能力

```
ESP32 能跑什么 AI？

✅ 可以（在算力范围内）：
  - 唤醒词检测 (WakeNet)：~300KB 模型，~30ms 推理
  - 命令词识别 (MultiNet)：~500KB 模型，~50ms 推理
  - VAD 语音活动检测：~10KB 模型，~5ms 推理
  - 简单图像分类：~200KB 模型，~100ms 推理
  - 手势识别（IMU数据）：~50KB 模型，~10ms 推理
  - 异常检测：~20KB 模型，~5ms 推理

❌ 不行（需要更强的芯片）：
  - 大语言模型 (LLM)：需要 GB 级内存
  - 实时视频理解：需要 GPU/NPU
  - 高精度语音识别 (ASR)：Whisper even tiny 需要几百MB
  - 人脸识别（高精度）：需要较强的 NPU
```

```c
#include "driver/gpio.h"
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"

#define LED_PIN GPIO_NUM_2  // 大多数 ESP32 开发板 LED 在 GPIO2

extern "C" void app_main(void) {
    gpio_reset_pin(LED_PIN);
    gpio_set_direction(LED_PIN, GPIO_MODE_OUTPUT);

    while (1) {
        gpio_set_level(LED_PIN, 1);
        ESP_LOGI("DEMO", "LED ON");
        vTaskDelay(pdMS_TO_TICKS(500));

        gpio_set_level(LED_PIN, 0);
        ESP_LOGI("DEMO", "LED OFF");
        vTaskDelay(pdMS_TO_TICKS(500));
    }
}
```

### 最小可行 Demo（MVD）

如果时间紧张，以下是最小可行的 Demo：

```
最小 Demo = ESP32-S3 + PDM 麦克风 + WiFi + Python 服务器

ESP32 端（~200 行代码）：
  1. WiFi 连接
  2. I2S PDM 麦克风采集
  3. WebSocket 发送音频
  4. 接收文字结果
  5. 串口打印结果

Python 服务器（~50 行代码）：
  import asyncio, websockets, openai

  async def handle(ws):
      while True:
          audio_data = await ws.recv()
          # 调用 OpenAI Whisper API
          text = openai.Audio.transcribe("whisper-1", audio_data)
          await ws.send(json.dumps({"type": "result", "text": text}))

  asyncio.run(websockets.serve(handle, "0.0.0.0", 8765))

成本：ESP32-S3 开发板 ¥50 + PDM 模块 ¥10 = ¥60
```
