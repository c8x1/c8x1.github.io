# 手机+无线麦克风协同拾音系统 — 创新技术评议报告设计

## 概述

为内部创新评议委员会撰写一份技术评议报告，论证"手机+无线麦克风协同拾音系统"的创新性和技术先进性。报告后续将用于支撑专利申请和技术秘密保护。

**报告受众**：公司内部创新评议委员会
**论证重点**：创新性（novelty）+ 技术先进性（technical advancement）
**不包含**：商业价值论证（由提案人自行完成）

## 系统定义

**场景**：主播/内容创作者佩戴 DJI Mic（无线领夹麦克风）拾取主说话人近场语音，同时使用手机内置 MIC 拾取环境音，两路信号在软件层协同处理，最终混合输出。

**核心问题**：
1. 手机 OS 如何同时采集两路异构音频源（内置 MIC + BT/Wireless 外部 MIC）
2. 两路信号存在未知且不稳定的时延差（冷启动延迟 + 持续漂移），如何对齐
3. 如何在主说话人说话时智能压低环境音（闪避），同时保留环境音的氛围感
4. 如何防止周围人说话误触发闪避

## 报告结构

采用**系统架构驱动**结构：按信号处理流水线的顺序组织，每个环节内嵌创新点论述。

**2.2 模块间信息流与反馈机制**（系统级创新核心论证）

3 条反馈回路：
1. **闪避→对齐**：闪避模块 SNR 估计反馈给 BFM 调整二值化阈值
2. **检测→校准**：声纹 VAD 置信度低于阈值时触发校准信号重播
3. **场景分类→全管线**：场景分类结果驱动所有模块参数自适应配置

**2.3 系统级创新声明**

论证"整体 > 部分之和"：单独看每个环节部分技术有先例，但在移动双路异构拾音场景下串联为闭环系统并通过模块间反馈实现自适应协同——无先例。

### §3 异构双路音频采集

**缺陷分析**：
- Android/iOS 无原生"同时打开两个异构输入源"标准化方案
- 现有无线麦克风只关注单路传输，不考虑与手机内置 MIC 协同
- 两路时钟域独立漂移，长时间录制延迟累积可达数百 ms
- BT 编解码切换造成延迟阶跃

**创新点**：

1. **异构双路采集中间件**：统一抽象层 + 双缓冲环形队列 + 滑动窗口采样率偏差实时估计（无需外部时钟参考）
2. **延迟指纹库**："设备型号 × BT 编解码器 × 实测延迟"映射表，首次连接查表粗估（±20ms），众包更新
3. **编解码切换无感恢复**：监测 codec 切换事件 → 冻结对齐状态 → 快速 Chirp 重校准 → 恢复 BFM 追踪

**对比表**：本方案 vs. 现有单路无线传输方案

### §4 环境自适应冷启动校准

**缺陷分析**：
- 传统校准信号参数固定，不考虑环境噪声水平
- 可听 Chirp 用户感知明显
- 近超声 Chirp 在嘈杂环境高频衰减严重
- 无方案能根据实时环境自动选择最优校准信号

**创新点**：

4. **噪声自适应校准信号（NACS）**：三级策略——安静(<35dBA)→近超声 / 中等(35-60dBA)→频谱整形 / 高噪(>60dBA)→高能量 LFM+语音叠加
5. **心理声学掩蔽感知校准**：利用播放侧音频掩蔽曲线，在掩蔽裕量内嵌入扩频校准码
6. **隐写式校准**：将 Chirp 编码到功能提示音（TTS "协同拾音已开启"）的共振峰间隙中
7. **多因子校准信号决策引擎**：四维矩阵（噪声 × 说话状态 × 播放状态 × 时间约束）→ 最优信号类型+参数

**对比表**：本方案 vs. 传统固定 Chirp

### §5 低复杂度持续延迟对齐

**缺陷分析**：
- 冷启动校准后延迟持续变化（BT 抖动 + 时钟漂移）
- GCC-PHAT 计算量大（~5 MIPS），不适合移动端持续运行
- BFM (Gao 2024) 计算量极低，但：单尺度、固定阈值、无置信度评估

**创新点**：

8. **多尺度二值化特征匹配（Multi-Scale BFM）**：三级金字塔（粗 10ms/±200ms → 中 2ms/±20ms → 精 亚采样/±2ms），搜索量 -80%
9. **环境自适应二值化阈值**：根据实时噪声 floor 动态调整，同时输出匹配置信度分数
10. **跨模态辅助 BFM（IMU + 音频联合估计）**：DJI Mic IMU 检测说话微振动作为辅助验证通道，极端噪声下兜底
11. **置信度驱动的被动/主动模式切换**：置信度 >0.8 被动追踪 / 0.5-0.8 警戒（超声 Chirp 频率↑）/ <0.5 主动校准

**对比表**：本方案 vs. 原始 BFM (Gao 2024)

### §6 目标说话人选择性检测

**缺陷分析**：
- 普通 VAD 无法区分目标/干扰说话人
- 声纹 VAD 需预注册，高噪环境准确率下降
- 现有方案均为单路处理，无法利用双路空间信息差异
- 佩戴式麦克风的 IMU/体传导信息被浪费

**创新点**：

12. **双路信号空间互验证 VAD**：DJI Mic ✅+手机 ✅ → 主播说话 / DJI ✅+手机 ❌ → 疑似误触发 / DJI ❌+手机 ✅ → 周围人 / 均 ❌ → 静默
13. **快速声纹注册 + 增量更新**：3 秒口令注册 → 运行时 <5ms 验证 → 增量学习适应嗓音变化
14. **多模态说话人存在检测**：IMU 微振动 + 呼吸气流模式 + 体传导声学三重验证，与环境噪声无关

**对比表**：本方案 vs. 普通 VAD vs. Google Personal VAD

### §7 智能音频闪避与自适应混音

**缺陷分析**：
- 传统 ducking 全频段宽带压低，环境音质感损失严重
- Ducking 深度固定，不随 SNR 变化
- 场景切换靠手动
- VAD 检测延迟 50-100ms，闪避启动有"硬切"感

**创新点**：

15. **频谱选择性闪避（Spectral-Selective Ducking）**：只对与主说话人频谱重叠的频段做增益衰减，保留不重叠频段的环境音
16. **SNR 自适应闪避深度**：sigmoid 映射 SNR→ducking depth（>20dB 浅 -6dB / 10-20dB 中 -12dB / <10dB 深 -18dB）
17. **场景感知混音策略引擎**：Vlog 模式 / 环境展示模式 / 对话模式 / 紧急感知模式（反向 ducking），500ms 交叉渐变
18. **IMU 预测性预闪避**：IMU 检测头部运动 → VAD 触发前 200ms 启动渐弱 → 消除 VAD 延迟感知

**对比表**：本方案 vs. 传统侧链压缩 vs. Sonible smart:comp（后期）
