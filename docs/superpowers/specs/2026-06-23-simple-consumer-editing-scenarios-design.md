## 1. 报告定位

- **与旧报告关系**：独立编号 C1-C7，附录映射回旧报告 S1-S8。两份报告各有定位（消费级 vs 专业级），可互相索引。

## 2. 场景集（C1-C7）

以**用户意图/任务**为主线（用户脑子里想做的一件事），每个场景用消费级需求定义、内容引向学术 SOTA 技术方向。

| # | 场景（消费级需求） | 学术 SOTA 技术方向（待填） | 映射旧场景 |
|---|---|---|---|
| C1 | 擦除与对象分离 | 音源分离 / 对象级擦除 / zero-shot separation | 新，近 S2/S7 |
| C2 | 一键降噪/增强修复 | 语音增强 / dereverb / inpainting | S2 |
| C3 | 文字改录音/口误秒改 | token infilling / 编辑即填空 | S1 |
| C4 | 片段裁剪拼接重组 | 音频拼接 / 结构编辑 | 新 |
| C5 | 轻量补一句/换一声 | 零样本 VC / overdub / 隐私匿名化 | S4+S1 |
| C6 | 口音归一/方言转标准 | 口音转换 / 模型编辑 / 流式 | S6 |
| C7 | 长内容局部修 | 长时段一致性 / 编辑端 | S8 |

**取舍说明**：
- C1（擦除/分离）单列为消费级旗舰动作（三星橡皮擦认知度最高）。
- C4（裁剪拼接）保留：与 C3（改字=改内容）不同，裁剪=改结构，是独立消费意图。
- C5 含隐私匿名化作为子诉求，不单列。
- 能力边界：后编辑为主 + 轻量生成（overdub/一句话克隆），不含纯 TTS/音乐生成。

## 3. 每场景内部结构（固定五段）

1. **场景痛点与用户诉求** — 消费级需求视角（普通用户拿到一段录音想干嘛、怕什么），列 3-4 个典型用户 + 共性诉求。
2. **学术 SOTA 与技术进展** — 本报告相对旧报告的增量重心：学术界在这个消费需求上聚焦什么方向、已解决到哪、路线怎么演进（2024-2026）。叙述性，引出下面的 paper。直接对应"在后编辑方向上学术界研究聚焦的方向以及正在解决的场景技术需求"。
3. **paper→问题映射** — SOTA 论文清单（arXiv ID 经核实），每条一句话"解决了什么消费级问题"。
4. **技术点** — 关键技术名词逗号串（与旧报告同风格）。
5. **成熟度与缺口** — 🟢🟡🔴 + 缺口标注，区分"领域真空白" vs "评测显示未就绪"（沿用旧报告的区分）。

**去掉**：旧报告的"落地情况"段（归同事工业界调研）。
**篇幅**：每场景约 30-50 行 MD（比旧报告略短，消费级诉求更聚焦）。
**核心差异**：旧报告把"现状"压得很短；本报告把第 2 段（学术 SOTA 与技术进展）展开成主体叙述。

## 5. 关键洞察（消费级视角）

在旧报告 5 条洞察基础上重写消费级读法，预期重点：
1. 消费级旗舰动作是 C1 擦除/分离（三星橡皮擦认知度最高），但学术上 zero-shot 通用分离仍弱。
2. 消费级"不可感知/局部性"要求比专业更严（普通用户不会调参数，要一键可靠）。
3. C7 长内容局部修仍是真空白（编辑端，一致性研究集中在生成端）。
4. 消费级需求-能力错配点（待研究阶段确认具体场景）。
