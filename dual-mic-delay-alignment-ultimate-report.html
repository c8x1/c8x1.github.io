为什么算法不能单独解决冷启动

时延估计需要两路里存在共同可观测信号。首个有效共同信号出现前，算法没有信息可以估计。主动校准的本质是：在正式内容开始前，主动制造一个高 SNR、形状已知的共同信号。

2. 问题本质与约束

⚡ 核心痛点是"冷启动"— TDE 算法需要语音信号才能检测延迟，第一句话开头没有对齐依据。精度要求：对齐后残余误差 <10ms（Haas Effect 阈值）。
维度内容
场景手机内置麦克风 + 外接无线麦克风（如 DJI Mic，标称延迟 ~15ms）双路拾音
核心痛点两路信号存在固定/时变延迟，混音后产生重音
精度要求对齐后残余误差 <10ms（人耳不可感知重音的阈值）
现有方案基于 Interspeech 发表的 TDE 算法做实时对齐
待解决问题TDE 需要语音信号才能检测延迟 → 冷启动问题
约束移动端实时运行；用户无感知或低感知；不依赖特殊硬件改造
为什么是 10ms？

10ms 对应空间距离约 3.4m（声速 343m/s）。时间差小于 10ms 时，人耳将它们感知为同一声源（优先效应 / Haas Effect）。实际中 5ms 以下更保险。

手机喇叭发探针时

estimated_delay = input_path_delta + speaker_to_mic_geometry_delta

喇叭播放延迟是两路共有的发声起点；真正进入估计的是两条输入链路差，以及喇叭到两支麦的空气传播路径差。

正式人声混音真正需要

wanted_delay = input_path_delta + mouth_to_mic_geometry_delta

如果喇叭位置和嘴巴位置差得很远，校准值会带几何偏差。声速约 343m/s，10ms 对应约 3.43m 路径差；20cm 位置误差约 0.6ms，通常可接受。

关键设计含义

手持手机拍摄时，用手机喇叭 chirp 校准通常足够。如果手机在桌上、DJI 夹在胸口、说话者频繁移动，最好改成"用户说开始/打响指"的口部源校准。

4. 主动校准信号

⚡ 快速结论：推荐首选：线性 Chirp (LFM) + Tukey 窗，50ms，200Hz→8kHz 扫频，处理增益 26dB。实现最简单，精度 <1ms，远超 10ms 要求。Golay 互补序列精度最高但不适合无线场景（需播放两次）。近超声方案体验最好但需硬件验证。

4.1 线性 Chirp（LFM） ⭐ 推荐

数学定义

x(t) = A · sin(2π·f₀·t + π·k·t²) t ∈ [0, T] f₀ = 起始频率 f₁ = 终止频率 B = f₁ - f₀ = 带宽 k = B / T = 扫频速率 T = 信号持续时间

匹配滤波检测

匹配滤波本质是将接收信号与已知 chirp 的时间反转共轭做卷积，等价于频域实现：

// 频域匹配滤波（高效实现）
C(f) = S(f) · X*(f)        // S=接收信号频谱, X=chirp模板频谱
c[k] = IFFT{C(f)}           // 互相关结果

// 处理增益 (Processing Gain)
G_p = T × B                  // 时间-带宽积

// 示例: T=50ms, B=7.8kHz (200Hz→8kHz)
G_p = 0.05 × 7800 = 390 ≈ 25.9 dB

处理增益 26dB 意味着：即使 chirp 信号的音量比环境噪声低 20dB，匹配滤波后仍然能清晰检测到峰值。

旁瓣问题与窗函数

线性 chirp 的匹配滤波输出有旁瓣（sidelobes）。矩形窗下第一旁瓣为 -13.2 dB（相对主瓣）。施加窗函数可抑制旁瓣：

窗函数第一旁瓣主瓣展宽适用场景
矩形（无窗）-13.2 dB1.0×SNR 优先
Hamming-42.7 dB1.5×通用

亚样本精度插值

互相关峰值落在离散样本上，但真实延迟可能在两个样本之间。使用对数域抛物线插值：

// k = 峰值样本索引, c[] = 互相关结果
α = log(|c[k-1]|)
β = log(|c[k]|)
γ = log(|c[k+1]|)

δ = (α - γ) / (2 · (α - 2β + γ))   // 亚样本偏移
τ = k + δ                             // 精确延迟（样本数）
delay_seconds = τ / fs
插值精度参考

对数域抛物线插值在 SNR > 10dB 时精度约 ±0.02 样本（±0.42μs @48kHz），远超 10ms 要求。在极低 SNR（<5dB）条件下，该估计器存在约 0.05-0.1 样本的系统偏差，如需更高精度可改用 Quinn 或 Macleod 估计器。

4.2 指数扫频 ESS（Farina 方法）

数学定义

x(t) = sin( K · (e^(t/L · ln(f₁/f₀)) - 1) ) K = 2πf₀L / ln(f₁/f₀), L = T

ESS 的关键特性：频率随时间指数增长，低频部分花费更多时间（能量更多），高频部分扫得更快。

逆滤波器归一化

x_inv(t) = (1/A_norm) · x(T - t) · e^(t/T · ln(f₁/f₀)) 其中 A_norm 为归一化常数，确保反卷积后脉冲响应的 DC 增益为单位值。 省略归一化会导致脉冲响应的绝对幅度取决于扫频参数，影响跨配置 SNR 比较。

非线性失真分离 — ESS 独有优势

如果手机扬声器在大音量下播放时产生非线性失真（削波），产生的各阶谐波失真在反卷积后会出现在主脉冲的负时间区域，可以干净地分离：

反卷积结果： ...[3次谐波]...[2次谐波].....[主脉冲(线性IR)]... ←── 负时间 ────────────────── 0 ─── 正时间 ──→ → 主脉冲不受谐波失真污染，延迟检测更可靠

ESS 的定位

如果你不仅想测时延，还想诊断两路通道的频率响应差异（理解为什么两路 mic 声音不同），ESS 是最佳选择。但对于"录音开始时的快速校准"，ESS 时长偏长（≥1s），Chirp 更适合。

4.3 最大长度序列（MLS）

数学基础

MLS 由线性反馈移位寄存器（LFSR）生成，序列长度 P = 2n - 1，值域 {+1, -1}。反馈多项式必须是本原多项式。

MLS 长度选择建议

对于 10ms 精度目标和 48kHz 采样率，n=10 (P=1023, 21ms) 是最实际的选择——30dB 增益足够，播放时间最短。如果对 SNR 有更高要求（嘈杂环境），升级到 n=12。

自相关特性

R_MLS[k] = 1 当 k = 0 (mod P)；-1/P 当 k ≠ 0 P=4095 时, -1/P ≈ -0.000244 → 近似完美 delta 函数

FHT 加速

MLS 互相关可用快速 Hadamard 变换（FHT）加速，计算量 O(N·log₂N) 且全是加减法，没有乘法：

4.4 Golay 互补序列

数学定义

Golay 互补序列是一对二值序列 (A, B)，满足：

R_A[k] + R_B[k] = 2N 当 k=0；0 当 k≠0 两个序列各自自相关的旁瓣完全抵消，合成完美 delta 函数。

优势：零旁瓣 + ~3dB 优于 MLS

Golay 对是唯一能做到非周期自相关完美零旁瓣的信号：两个序列的自相关函数相加后，所有非零 shift 处的旁瓣精确抵消为零。SNR 约 3dB 优于同等长度的 MLS。

Golay 的实际时间开销 — 不推荐无线场景

两段序列必须顺序播放，且中间需要保护间隔（guard interval）防止混响重叠。以 N=2048 为例：每段 42.7ms + 保护间隔 ~15ms → 总校准时间 ~100ms，是 Chirp 50ms 的两倍。

更关键的是，两次播放间如果无线麦克风延迟发生变化（常见于蓝牙重传），会导致不对称相关峰，严重影响精度。结论：不推荐用于无线麦克风场景。

5. 音频水印持续校准

⚡ 快速结论：音频水印将同步标记嵌入正常音频流中，不仅解决冷启动，还能持续跟踪延迟漂移。扩频水印(DSSS) + 导频音(Pilot Tone) 组合是最全面的方案，但实现复杂度较高。适合作为 Phase 3 的高级方案。

5.1 核心原理差异

方案 A：播放 → 等待 → 检测（一次性或周期性） → 校准信号和用户信号在时间上是分离的 方案 B：水印信号与用户信号始终共存 → 校准信息嵌入在正常音频流中 → 任何时候都可以提取对齐信息 → 不仅解决冷启动，还能持续、无缝地跟踪延迟变化

5.2 扩频水印（DSSS）

基本原理

借用直接序列扩频通信（DSSS）思想：将同步标记的能量分散到很宽的频带，每个频点的能量极低（人耳感知不到），但用同一个 PN 序列做相关可以恢复出来。

嵌入算法（频域，利用心理声学掩蔽）

def embed_watermark_freq(host_audio, pn_seq, alpha, fs):
    """频域扩频水印嵌入"""
    frame_size = 2048   # 42.7ms @48kHz
    hop_size   = 512    # 10.7ms
    frames = stft(host_audio, frame_size, hop_size)

    for i in range(frames.shape[0]):
        mag   = np.abs(frames[i])
        phase = np.angle(frames[i])

        # 心理声学掩蔽阈值（简化版）
        # 每个频点的嵌入强度不超过该频点的掩蔽阈值
        mask = compute_masking_threshold(mag, fs)

        # 水印 = PN序列 × 嵌入强度 × 掩蔽阈值
        wm_spectrum = alpha * pn_seq * mask

        # 叠加
        frames[i] = (mag + wm_spectrum) * np.exp(1j * phase)

    return istft(frames, frame_size, hop_size)

心理声学掩蔽模型（HAS）

水印必须听不到，嵌入强度受限于人耳听觉系统的掩蔽阈值：

G_ss = 10·log₁₀(L) dB (L = PN序列长度 = 每帧频点数量) L=1024: 30.1 dB | L=2048: 33.1 dB | L=4096: 36.1 dB

检测算法（相关接收机）

def detect_watermark_delay(y1, y2, pn_seq, frame_size, hop_size):
    """检测两路麦克风拾取的水印，计算时延差"""
    frames1 = stft(y1, frame_size, hop_size)
    frames2 = stft(y2, frame_size, hop_size)

    # 每帧与 PN 序列做点积（相关）
    corr1 = [np.dot(np.abs(f[:len(pn_seq)]), pn_seq) for f in frames1]
    corr2 = [np.dot(np.abs(f[:len(pn_seq)]), pn_seq) for f in frames2]

    # 两路相关序列的互相关 → 帧级时延差
    lag_corr = np.correlate(corr1, corr2, mode='full')
    peak_lag = np.argmax(np.abs(lag_corr)) - len(corr2) + 1

    return peak_lag * hop_size / fs  # 秒

5.3 导频音（Pilot Tone）

原理

在特定频率持续插入一个低幅度单音，两路麦克风拾取后通过 Hilbert 变换提取瞬时相位，相位差 = 延迟差。

def detect_pilot_delay(y1, y2, pilot_freq, fs):
    """导频音相位差 → 时延"""
    # 带通滤波提取导频频段
    p1 = bandpass(y1, pilot_freq - 50, pilot_freq + 50, fs)
    p2 = bandpass(y2, pilot_freq - 50, pilot_freq + 50, fs)

    # Hilbert 变换 → 瞬时相位
    phase1 = np.unwrap(np.angle(hilbert(p1)))
    phase2 = np.unwrap(np.angle(hilbert(p2)))

    # 相位差 → 时延
    delta_phase = np.mean(phase1 - phase2)
    return delta_phase / (2 * np.pi * pilot_freq)

精度分析

δt = δφ / (2π·f_pilot) 例: f_pilot=18kHz, SNR=10dB → δφ≈0.22rad → δt≈2.0μs SNR=0dB → δφ≈0.71rad → δt≈6.3μs 精度极高，远超 10ms 要求

相位模糊问题与多频 Vernier 解法

单频导频有严重的相位模糊

fpilot=18kHz → 不模糊范围 = 1/18000 = 55.6μs。DJI Mic 延迟约 15ms → 15ms/55.6μs ≈ 270 个周期 → 严重模糊。

多频 Vernier 解法：使用两个导频频率 f₁, f₂：

等效不模糊范围 = f₁·f₂ / |f₁ - f₂| 例: f₁=16kHz, f₂=16.5kHz → 不模糊范围 = 16000×16500/500 = 528 ms 完全可以覆盖 DJI Mic 的 ~15ms 延迟。 如需更大范围，加第三频 f₃=15kHz → 与 f₁ 的差频 1kHz → 不模糊范围 1s

6. 被动 TDE 基线 — GCC-PHAT

⚡ 快速结论：GCC-PHAT 是双麦克风时延估计的行业标准基线。完全被动、零用户感知、计算量极低，但需要持续声源——这正是冷启动问题所在。它的角色是"持续精细对齐引擎"，与方案 A/B 互补而非替代。

6.1 原理

广义互相关-相位变换（Generalized Cross-Correlation with Phase Transform）：

τ̂ = argmax_τ ∫₋π^π [X₁(ω)·X₂*(ω) / |X₁(ω)·X₂*(ω)|] · e^(jωτ) dω PHAT 加权将所有频率分量等权化，抑制噪声频谱的影响， 在混响和噪声环境下比基本互相关更鲁棒。

6.2 优势与局限

维度说明
✅ 无需播放信号完全被动，零用户感知
✅ 计算量极低一次 FFT + PHAT 加权 + IFFT
✅ 抗混响能力强PHAT 加权抑制混响导致的频谱着色
❌ 需要持续声源静默期间无信号 → 估计不稳定（冷启动问题）
❌ 精度受限于带宽窄带信号精度差

7. 检测算法与亚采样精度

⚡ 匹配滤波 + 对数域抛物线插值是最佳组合— 在 SNR>10dB 时精度约 ±0.02 样本（±0.42μs @48kHz），远超 10ms 要求。多径环境下取"最早且足够强的峰"而非全局最大峰。

7.1 匹配滤波检测流程

手机扬声器播放已知校准信号 s(t)，两路麦克风分别拾取后对原始信号做互相关（匹配滤波），找到各自的到达时间，两者之差即为时延差。

播放校准信号 s(t) │ ┌─────────┼─────────┐ ↓ ↓ 手机 mic 录音 DJI Mic 录音 r₁(t) r₂(t) │ │ ↓ ↓ 匹配滤波/相关 匹配滤波/相关 p₁(τ) = r₁⋆s p₂(τ) = r₂⋆s │ │ ↓ ↓ 峰值检测 峰值检测 τ₁ = argmax(p₁) τ₂ = argmax(p₂) │ │ └─────────┬─────────┘ ↓ Δτ = τ₁ − τ₂ ← 两路时延差

因为 s(t) 是自定义信号，其自相关特性可以被优化到极其尖锐（近似 delta 函数），互相关峰的 SNR 远高于被动方案。

7.2 亚采样插值方法

互相关峰值落在离散样本上，但真实延迟可能在两个样本之间。需要亚采样插值才能达到 <1ms 的精度。

方法SNR=20dBSNR=10dB复杂度推荐度
无插值±0.5 样本±0.5 样本低基础

对数域抛物线插值公式

设峰值在 index k，互相关结果 c[] α = log(|c[k-1]|) β = log(|c[k]|) γ = log(|c[k+1]|) δ = (α - γ) / (2 · (α - 2β + γ)) // 亚样本偏移 τ = k + δ // 精确延迟（样本数） delay_seconds = τ / fs
插值精度与偏差说明

对数域抛物线插值在 SNR > 10dB 时精度约 ±0.02 样本（±0.42μs @48kHz），远超 10ms 要求。在极低 SNR（<5dB）条件下存在约 0.05-0.1 样本的系统偏差，如需更高精度可改用 Quinn 或 Macleod 估计器。对于本项目的 10ms 精度目标，对数域抛物线插值完全足够。

参考：Lai et al., NTNU

7.3 多径/混响处理

室内环境下，校准信号会通过多条路径到达两路 mic。互相关输出会出现：

  • 主峰：直达路径（最强或最早）
  • 次峰：反射路径（较弱、较晚，通常在主峰后 5-50ms）

桌面、墙面、手持姿态都可能带来反射峰。如果简单地取全局最大峰，可能选中反射路径而非直达路径，导致时延估计偏差。

策略：取最早出现的显著峰（而非全局最大峰）

对互相关结果做 onset detection：找到超过噪声基底 6dB 以上的第一个峰。如果两路 mic 到声源的距离差很小，反射路径通常在主峰之后出现——直达峰仍然清晰可辨。

此外，建议重复测量 3 次并取中位数（median），以降低偶发噪声和反射峰误选的概率。

7.4 置信度门控

校准结果必须通过置信度检查，才能用于正式混音。以下为关键门控条件：

置信度门控清单
  • 相关峰 peak / second_peak 大于阈值（如 1.5-2.5）
  • 两路都能检测到探针，且峰值能量高于噪声底
  • 估计 delay 落在设备先验范围内（如 -200ms 到 +200ms）
  • 重复探针的 delay 方差小于阈值（如 1-2ms）
  • 失败时不要硬用错误值：降副麦、单主麦，或进入 fallback

8. 心理声学与用户体验设计

⚡ 最佳用户体验：将校准 Chirp 嵌入"录音开始"提示音或倒计时音效中— 用户听到的是自然提示音，算法看到的是可定位探针。校准信号段保留在预滚缓存中，不写入最终素材。

8.1 三种频谱整形策略

用户体验的关键——校准信号不能让用户觉得刺耳或突兀。以下三种策略各有取舍：

策略 A：高频偏好

将校准信号集中在 8-16kHz 频段。成年人对 >12kHz 的感知显著下降，年轻人感知略好但总体弱于中低频。

代价：带宽减半，但时延分辨率仍有 ~140μs。

策略 B：宽带 + 低振幅 + 渐变

200Hz-16kHz 宽带，振幅 30-40% 满量程，前后 10ms 余弦淡入淡出。

总感知约 50-55 dB SPL，接近正常对话音量——用户会注意到但不刺耳。

策略 C：掩蔽效应嵌入 ⭐最佳UX

利用 UI 提示音的前向掩蔽窗口（~20ms），将校准信号紧跟在提示音之后播放，感知降低 10-20dB。

用户几乎无感知，算法仍可定位。来源：Amazon Science

8.3 产品化策略

产品默认策略

默认播放一个短、可解释的 UI 声音，其中嵌入 2-8kHz log chirp。两路麦克风分别对已知探针做匹配滤波，取相关峰位置相减，得到输入链路相对 delay。探针段保留在预滚缓存里，但不写入最终素材。

失败兜底策略

9. 关键工程约束

⚡ 三个最容易被忽略的工程陷阱：1) 蓝牙编码延迟不是固定的，抖动可达 ±5-20ms；2) 独立时钟漂移 ~0.1ms/s，100 秒后累积 10ms；3) 手机 DSP（AEC/NS/AGC）会主动消除校准信号。必须用 UNPROCESSED/measurement 模式获取原始音频。

9.1 蓝牙编码延迟抖动

原报告将 DJI Mic 的延迟视为固定偏移（~15ms），但实际蓝牙编码延迟是时变的：

即使初始校准完美，延迟可能在录制过程中漂移 ±5-20ms — 超出 10ms 精度要求。必须实现自适应跟踪（连续或半连续），而非离散重校准事件。增加看门狗机制：如果两次校准之间延迟变化 >5ms，立即触发重校准。检查 DJI Mic 是否暴露了编码协商参数（如 codec 类型、bitpool），可用于约束延迟变化范围。

9.2 时钟漂移

手机和无线麦克风有独立的采样时钟，即使蓝牙延迟完全稳定，两个 ADC 的采样率差异也会导致对齐逐渐漂移：

典型消费级晶振精度: ±20-50 ppm 最坏情况 (两端各 50ppm, 反向偏差): 相对偏差 = 100 ppm = 0.01% @48kHz: 4.8 样本/秒 漂移 = 0.1 ms/s → 10 秒后累积 1ms，100 秒后 10ms 典型情况 (两端各 20ppm): → ~40 秒后累积 1ms，400 秒后 10ms
时钟漂移是方案 B（持续水印）的核心价值所在

对于超过 20 秒的录音，持续或周期性重估计是必须的，与校准方法无关。方案 B 的扩频水印天然提供持续跟踪能力，这是它作为长期方案的根本优势之一。方案 A 的一次性校准只能解决冷启动，无法应对时钟漂移 — 必须搭配持续跟踪机制。

9.3 手机音频 DSP 管线

现代手机对麦克风输入施加了激进的 DSP 处理，这些处理专为去除"非语音信号"设计，会主动抑制或破坏校准信号和水印：

DSP 模块行为对水印的影响

自动增益 (AGC)动态调整增益水印幅度不稳定
回声消除 (AEC)去除扬声器回放信号致命！ 可能直接消除校准信号

关键对策
  1. 获取原始音频：通过平台特定 API 绕过 DSP 处理
    • Android: AudioSource.UNPROCESSED 或 VOICE_RECOGNITION 模式
    • iOS: AVAudioSession.Mode.measurement（禁用 AEC/NS/AGC）
  2. 关闭 AEC：如果 App 同时播放校准信号和录音，必须确保 AEC 不会消除校准信号
  3. 测试水印存活率：通过常见 DSP 链（WebRTC NS, RNNoise, 厂商特定）测试水印是否存活
  4. 频带选择：将水印放在 NS 不太激进的频段（通常 6-8kHz 边缘），或使用频率跳变避免窄带抑制器

9.4 多径反射

桌面、墙面、手持姿态都可能带来反射峰。如果简单地取全局最大峰，可能选中反射路径而非直达路径，导致时延估计偏差。室内环境下校准信号通过多条路径到达两路 mic，互相关输出会出现主峰（直达路径）和次峰（反射路径，通常在主峰后 5-50ms）。

策略：取"最早且足够强"的峰

对互相关结果做 onset detection：找到超过噪声基底 6dB 以上的第一个峰，而非绝对最大峰。此外，建议重复探针取 median（3 次），以降低偶发噪声和反射峰误选的概率。

9.5 不要把主动探针当唯一真相

探针只测量探针声源位置下的相对 delay

主动探针估计的是"探针声源位置下的相对 delay"。正式说话开始后，仍应把语音 TDE 作为二次确认和微调来源。产品上可以把主动校准视为首句保护层，把持续语音估计视为长期稳定层。

10. 持续跟踪与漂移对策

⚡ 初始对齐后仍会漂移（时钟 + 蓝牙）：三条持续跟踪路线 — 扩频水印（每 500ms 更新）、NLMS 自适应滤波器（语音段持续适应）、BFM 二值化特征匹配（极低计算量）。搭配看门狗：延迟变化 >5ms 时立即重校准。

10.1 扩频水印持续跟踪

一次性校准解决"录音开始时"的冷启动问题。如果时延在录音过程中漂移（蓝牙编码抖动、时钟漂移），扩频水印可以提供持续、无缝的跟踪：

参数推荐值
水印信号CSS chirp, 12-18kHz, 重复周期 500ms
水印功率低于内容信号 25-30dB
检测方法滑动互相关，每 500ms 更新
跟踪精度±2-5ms

来源：MDPI 2024 "Audio Watermarking System in Real-Time" + Amazon Science 感知水印论文

10.2 NLMS 自适应滤波器

初始对齐完成后启动 NLMS 自适应滤波器，在语音段持续适应漂移变化：

// NLMS 自适应滤波器跟踪流程
初始化:  用校准信号的时延值设置初始补偿延迟
运行:    NLMS 滤波器在语音段持续适应
检测:    每 100ms 检查滤波器系数峰值位置
          如果偏移 > 阈值(2ms) → 更新补偿
重校准:  如果滤波器发散 → 重新播放校准信号

来源：XMOS ADEC (XVF3610) 自适应时延估计完整实现

10.3 BFM 二值化特征匹配

Interspeech 2024 Gao & Su (Tencent) 提出的 Binary Feature Matching (BFM) 方法：将音频特征二值化后用 XOR 位运算匹配，计算量极低，适合作为移动端持续追踪器。BFM-FP（音频指纹）变体可在任何音频内容上工作（含环境噪声），不需要专门的校准信号。

10.4 周期性重校准 + 看门狗

看门狗机制

如果连续两次校准延迟差 >5ms → 立即触发一次 Chirp 重校准。每 10s 播放超声 Chirp Burst（20ms, 16-19kHz）持续跟踪漂移（蓝牙抖动 + 时钟漂移），在语音间歇期也可校准。

11. 实施与实验路线

⚡ 建议分 4 个阶段实施：Phase 1 可听 Chirp（1-2周，最快上线）→ Phase 2 近超声升级（需硬件验证）→ Phase 3 扩频水印（1-3月）→ Phase 4 产品级优化。每个阶段都有明确的成功标准和失败处理。

11.1 Phase 时间线

Phase 1（1-2 周）: 可听 Chirp 校准 P0 最快上线
  • 功能开启 → 播放语音提示"协同拾音已开启" → 尾部叠加 50ms LFM Chirp (200Hz→8kHz)
  • 匹配滤波 + 对数域抛物线插值
  • 确保获取 UNPROCESSED/measurement 模式的原始音频
  • 验证精度 <10ms（至少 10 次重复测量，标准差 <2ms）
Phase 2（2-4 周）: 近超声升级 + 延迟跟踪 需硬件验证
  • 在目标设备矩阵上测试 16-20kHz 频率响应
  • SNR >6dB → 升级为近超声 Chirp
  • SNR <6dB → 保持可听 Chirp
  • 增加周期性校准 + 看门狗
  • 运行时回退协议
Phase 3（1-3 月）: 扩频水印持续校准 P2 高级版
  • 手机端音频输出流中嵌入扩频水印
  • 频段: 2-8kHz（抗蓝牙编码）+ 16-19kHz（近超声辅助，如硬件支持）
  • 自相关扩频检测做持续延迟跟踪
  • 与 GCC-PHAT 做互补融合
Phase 4（3-6 月）: 产品级优化
  • 心理声学自适应嵌入
  • 多设备延迟预测模型
  • A/B 测试不同校准策略

11.3 方案组合推荐

功能开启时： 1. 播放语音提示："协同拾音已开启"（~1.5秒） 2. 语音提示最后 50ms，叠加 Chirp 信号（2kHz→8kHz, -15dBFS） 3. Chirp 结束后立即开始录音 4. 两路 MIC 做匹配滤波 → 得到初始时延 τ₀ 5. 用 τ₀ 做初始对齐 6. 切换到 GCC-PHAT / Interspeech TDE 做持续精细对齐 7. 每 10s 播放超声 Chirp Burst（20ms, 16-19kHz）持续跟踪漂移 8. 看门狗：连续两次校准延迟差 >5ms → 立即重校准
